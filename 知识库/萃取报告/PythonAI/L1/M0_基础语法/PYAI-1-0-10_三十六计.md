## 详细教学流程

### 步骤1：知识讲解（15分钟）
`#中负荷-操练` `#IFC-即时`

**教师话术**
> "上节课我们学了if，今天我们要学更强大的分支结构！
>
> **if-else：二选一**
> ```python
> if enemy:
>     hero.attack(enemy)
> else:
>     hero.moveRight()
> ```
> 有敌人就打，没敌人就走。只会执行其中一个！
>
> **elif：多选一**
> ```python
> if enemy.type == 'munchkin':
>     hero.attack(enemy)      # 小怪直接打
> elif enemy.type == 'ogre':
>     hero.shield()           # 大怪先防御
> else:
>     hero.moveRight()        # 其他情况走开
> ```
> elif是'else if'的缩写，可以有很多个！
>
> **嵌套if：条件套条件**
> ```python
> if enemy:
>     if enemy.health < 20:
>         hero.attack(enemy)  # 血少直接打
>     else:
>         hero.shield()       # 血多先防御
> ```"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 混淆if和elif | 用流程图解释 | 画出分支流程图，标明执行路径 |
| 嵌套层级混乱 | 强调缩进 | 每层嵌套多缩进4个空格 |
| 条件顺序错误 | 讲解优先级 | if/elif按顺序判断，第一个True就执行 |

### 步骤2：闯关实践（35分钟）
`#高负荷-产出` `#IFC-即时`

**教师话术**
> "现在进入'背靠背'关卡！
>
> 这一关有多种敌人，你需要：
> 1. 判断敌人类型，采取不同策略
> 2. 判断敌人血量，决定攻击还是防御
> 3. 判断敌人距离，决定是否需要移动
>
> 提示：用elif处理不同敌人类型，用嵌套if处理复杂条件！"

**即时反馈检查点**
- 检查if/elif/else的缩进是否一致
- 检查条件判断的顺序是否合理
- 检查是否有遗漏的情况（else兜底）

**核心代码模式**
```python
while True:
    enemy = hero.findNearestEnemy()

    if enemy:
        # 根据敌人类型选择策略
        if enemy.type == 'munchkin':
            # 小怪：直接攻击
            hero.attack(enemy)
        elif enemy.type == 'thrower':
            # 投掷手：先躲避再攻击
            hero.shield()
            hero.attack(enemy)
        elif enemy.type == 'ogre':
            # 食人魔：判断血量
            if enemy.health < 50:
                hero.attack(enemy)
            else:
                hero.shield()
        else:
            # 其他敌人：默认攻击
            hero.attack(enemy)
    else:
        # 没有敌人：移动寻找
        hero.moveRight()
```

**禁忌提醒**
- elif不能单独使用——必须跟在if后面
- else后面不能加条件——`else enemy.type == 'ogre'`是错的
- 嵌套太深难以阅读——超过3层考虑重构
- 条件顺序不当——把最常见的情况放前面

## 核心知识点

- if-else：二选一分支结构
- elif：多选一分支结构
- 嵌套if：条件中的条件

**底层逻辑**
- **决策树思维**：复杂问题分解为多个判断
- **完备性思考**：考虑所有可能情况，用else兜底
