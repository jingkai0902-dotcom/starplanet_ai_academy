## 详细教学流程

### 步骤1：知识讲解（15分钟）
`#中负荷-操练` `#IFC-即时`

**教师话术**
> "之前我们的代码是'傻傻地'按顺序执行。今天我们要让代码变聪明——学会判断！
>
> **条件语句 if**：如果...就...
> ```python
> enemy = hero.findNearestEnemy()
> if enemy:
>     hero.attack(enemy)
> ```
>
> 这段代码的意思是：先找最近的敌人，如果有敌人，就攻击它！
>
> **布尔值**：True（真）和 False（假）。if后面的条件结果是True就执行，False就跳过。
>
> **pass占位符**：暂时不知道写什么，先用pass占个位置。"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 混淆`=`和`==` | 口诀记忆 | "一个等号是赋值，两个等号是比较" |
| if后忘记缩进 | 用颜色标记 | 在编辑器中显示缩进标记 |
| 不理解布尔值 | 用开关类比 | "True是开，False是关" |

**禁忌提醒**
- 赋值`=`和比较`==`混淆——`if x = 5` 会报错，应该是 `if x == 5`
- if后面忘记冒号——`if enemy` 后面必须加冒号
- 缩进不一致——if下面的代码必须统一缩进

---

### 步骤2：闯关实践（35分钟）
`#高负荷-产出` `#IFC-即时`

**教师话术**
> "现在进入平原森林保卫战关卡！
>
> 这一关你需要：
> 1. 用if判断是否有敌人
> 2. 有敌人就攻击，没敌人就移动
> 3. 用pass占位，先搭好代码框架
>
> 提示：先写if的框架，再填充具体动作！"

**即时反馈检查点**
- 检查if后是否有冒号
- 检查条件表达式是否正确
- 检查缩进是否一致

**核心代码模式**
```python
while True:
    enemy = hero.findNearestEnemy()

    if enemy:
        # 有敌人，攻击
        hero.attack(enemy)
    else:
        # 没敌人，先用pass占位
        pass
```

## 核心知识点

- 条件语句：`if` 的结构（冒号、缩进）
- 布尔值：True/False
- 占位符：`pass` 语句在代码结构中的占位作用

**底层逻辑**
- **判断思维**：让程序根据条件做出不同反应
- **框架先行**：先搭结构再填内容的编程习惯
