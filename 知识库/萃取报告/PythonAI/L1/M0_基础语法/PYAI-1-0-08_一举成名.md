## 详细教学流程

### 步骤1：知识讲解（15分钟）
`#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天是综合挑战！我们要进入WAKKA MAUL竞技场，和其他玩家的代码对战！
>
> **竞技场规则**：
> - 你的英雄会自动执行你写的代码
> - 打败敌人获得金币
> - 金币可以升级装备
> - 代码越聪明，胜率越高！
>
> **综合运用**：
> ```python
> while True:
>     enemy = hero.findNearestEnemy()
>     if enemy:
>         hero.attack(enemy)
>     else:
>         hero.moveRight()
> ```
>
> 这段代码的逻辑是：一直循环，有敌人就打，没敌人就往右走。"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 代码逻辑混乱 | 画流程图 | 先用流程图理清思路再写代码 |
| 总是被打败 | 分析对手 | 观察对手策略，针对性优化 |
| 想要更强策略 | 引入优先级 | 先打血少的敌人，或先捡金币 |

### 步骤2：闯关实践（35分钟）
`#高负荷-产出` `#IFC-即时`

**教师话术**
> "现在进入竞技场！
>
> 任务清单：
> 1. 写一个基础的战斗循环
> 2. 观察战斗结果，分析问题
> 3. 优化代码，提高胜率
> 4. 和同学的代码对战！
>
> 策略提示：
> - 可以先找最近的敌人
> - 可以判断敌人血量
> - 可以在没敌人时收集金币"

**即时反馈检查点**
- 检查while循环是否正确
- 检查if条件是否合理
- 检查代码是否有死循环风险

**核心代码模式**
```python
while True:
    # 找最近的敌人
    enemy = hero.findNearestEnemy()

    # 找最近的金币
    coin = hero.findNearestItem()

    # 策略：有敌人先打敌人，没敌人捡金币
    if enemy:
        hero.attack(enemy)
    elif coin:
        hero.moveXY(coin.pos.x, coin.pos.y)
    else:
        hero.moveRight()
```

**禁忌提醒**
- 不要写死循环里没有任何动作——会导致程序卡死
- 不要忘记else分支——没有敌人时英雄会傻站着
- 不要只顾攻击忘记移动——可能被远程敌人打死

## 核心知识点

- 综合应用：while循环 + if条件 + 变量
- 策略设计：根据游戏规则设计最优策略
- 代码优化：通过测试反馈改进代码

**底层逻辑**
- **竞技学习**：通过对战激发优化动力
- **迭代思维**：代码不是一次写好的，需要不断优化
