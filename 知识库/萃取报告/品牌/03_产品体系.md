## 3.1 课程架构（阶梯式赋能）

### 场景：向家长介绍课程体系

**核心内容**

| 层级 | 年龄段 | 课程 | 核心目标 |
|------|--------|------|----------|
| 认知层 | 2.5-6岁 | STEM幼儿科创 | 感官探索、科学启蒙 |
| 思维层 | 6-12岁 | CODE少儿编程、VEX GO | 计算思维构建 |
| 技术层 | 9-15岁 | Python AI、C++ | 工具链掌握 |
| 创造层 | 10-18岁 | iDEALAB、星球研学 | 跨学科创新 |

**底层逻辑**
"认知-思维-技术-创造"四层递进模型，符合布鲁姆教育目标分类学：从记忆理解到应用分析，再到评价创造。

## 3.2 四大核心成长路径

### 场景：为家长规划学习路径

**核心内容**

| 路径 | 方向 | 课程链条 | 出口 |
|------|------|----------|------|
| 路径A | 信奥 | STEM → CODE → C++ | 算法竞赛/升学 |
| 路径B | 机器人 | STEM → CODE → VEX | 工程竞技/世锦赛 |
| 路径C | AI | STEM → CODE → Python | 人工智能/应用 |
| 路径D | 科创 | STEM → CODE → IDEA | 发明/专利 |

**底层逻辑**
"多出口"策略：不同孩子有不同的兴趣和天赋，提供多元化的发展路径，避免"一刀切"。每条路径都有明确的升学/竞赛出口，让家长看到投入的回报。
