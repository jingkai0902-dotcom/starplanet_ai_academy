CODE2-1是二年级的开篇模块，从CODE1的基础编程升级到传感器驱动的智能应用。学生将学习如何让程序"感知"环境并做出智能响应。

**单元教学策略**
- 视觉交互（01）：视频侦测，建立"程序能看见"的概念
- 基础传感器（02-05）：触碰开关、红外传感器、声音/倾斜传感器
- 高级传感器（06-10）：超声波测距、温度感知、RGB控制、电位器
- 执行器控制（11-12）：舵机精确角度控制

### 第1课：会跳舞的火柴人

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-1-01 |
| **课程名称** | 会跳舞的火柴人 |
| **认知负荷** | 中-高 |
| **核心技能** | 视频侦测、阈值判断、透明度特效 |

#### 详细教学流程

**步骤1：课堂问候（2分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们好！今天我们要让电脑'看见'我们的动作！你们相信吗？电脑真的能看见我们在动！"

---

**步骤2：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "【播放动画】斯斯发现了一个神奇的火柴人，它会跟着人的动作一起跳舞！这是怎么做到的呢？原来电脑有一双'眼睛'——摄像头，它能看见我们的动作！"

---

**步骤3：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "Scratch有一个神奇的功能叫'视频侦测'。它能检测摄像头前面有没有东西在动。"
>
> "视频侦测会给出一个数字（0-100）：
> - 0 表示完全没有动
> - 100 表示动得很厉害
> - 我们可以设置一个'门槛'，比如大于20就算在动"
>
> "还有一个小技巧：把视频透明度设为50%，这样既能看到自己，又能看到角色！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 摄像头没反应 | 检查权限 | 浏览器需要允许摄像头访问 |
| 太敏感/不敏感 | 调整阈值 | 从20调整到10或30 |
| 想增加难度 | 多角色响应 | 不同部位触发不同角色 |

**禁忌提醒**
❌ 不要在光线太暗的环境——视频侦测效果差
❌ 不要让学生站太远——检测不到动作
❌ 不要忘记开启摄像头——程序不会报错但没反应

---

**步骤4：编程设计（25分钟）** `#高负荷-产出` `#IFC-即时`

**教师话术**
> "现在我们来编程：
> 1. 打开视频侦测（将视频设为开启）
> 2. 设置透明度为50%
> 3. 重复执行：
>    - 如果（相对于角色的视频运动 > 20）
>    - 那么：下一个造型（让火柴人跳舞）"

**核心代码模式**
```
当绿旗被点击
将视频设为 [开启]
将视频透明度设为 (50)
重复执行
  如果 <(相对于 [角色] 的视频运动) > (20)> 那么
    下一个造型
    等待 (0.1) 秒
  结束
结束
```

---

**步骤5：家长班会（10分钟）** `#低负荷-热身` `#IFC-复盘`

**教师话术**
> "各位家长大家好！今天孩子们学习了'视频侦测'技术。这是计算机视觉的入门概念——让程序能够'看见'并响应真实世界的动作。这种技术在体感游戏、智能监控中都有应用！"

---

**核心知识点**
- 视频侦测：检测摄像头前的运动（0-100）
- 阈值判断：设定触发条件（如 > 20）
- 透明度控制：50%让视频和角色都可见

**底层逻辑**
- **计算机视觉入门**：程序可以"看见"真实世界
- **阈值概念**：数值比较决定是否触发动作
- **人机交互**：用身体动作控制程序

### 第2课：电路的秘密

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-1-02 |
| **课程名称** | 电路的秘密 |
| **认知负荷** | 中 |
| **核心技能** | 触碰开关、电路闭合原理、条件判断 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们家里的灯是怎么打开的？对，按开关！今天我们要了解开关背后的秘密——电路！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "电路就像一条路，电流要从这头走到那头。但如果路断了，电流就过不去了。
>
> 开关的作用就是'断路'和'通路'：
> - 开关断开 = 路断了 = 灯不亮
> - 开关闭合 = 路通了 = 灯亮了
>
> 我们的触碰开关就是这个原理！当两个金属片碰在一起，电路就通了。"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 不理解电路 | 用水管类比 | 电流像水，开关像水龙头 |
| 触碰不灵敏 | 检查连接 | 确保金属片接触良好 |
| 想增加功能 | 添加声音 | 开灯时播放"叮"的声音 |

**核心代码模式**
```
当绿旗被点击
重复执行
  如果 <触碰开关被按下?> 那么
    将 [灯] 设为 [开]
    将灯的造型设为 [亮]
  否则
    将 [灯] 设为 [关]
    将灯的造型设为 [暗]
  结束
结束
```

---

**步骤3：家长班会（10分钟）** `#低负荷-热身` `#IFC-复盘`

**教师话术**
> "各位家长好！今天孩子们学习了电路的基本原理。通过触碰开关，他们理解了'通路'和'断路'的概念。这是电子学的基础知识，为后续学习更复杂的传感器打下基础。"

---

**核心知识点**
- 电路闭合原理：通路导电，断路不导电
- 触碰开关：物理接触控制电路通断
- 条件判断：根据开关状态执行不同操作

### 第3课：商场自动门

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-1-03 |
| **课程名称** | 商场自动门 |
| **认知负荷** | 中-高 |
| **核心技能** | 红外避障传感器、条件循环、状态切换 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们去过商场吗？商场的门是不是很神奇？人走过去它就自动打开，人走了它又自动关上。今天我们就来揭秘自动门的原理！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "自动门的秘密是'红外传感器'！
>
> 红外传感器会发出一束看不见的光（红外线）。当有人走过来，红外线被挡住了，传感器就知道'有人来了'！
>
> 工作原理：
> - 红外线没被挡住 → 返回0 → 门关着
> - 红外线被挡住 → 返回1 → 门打开
>
> 我们还要加一个'延时关门'功能：人走了之后，等2秒再关门，这样更安全！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 传感器不灵敏 | 调整距离 | 手要在传感器正前方10cm内 |
| 门开关太快 | 添加动画 | 用多个造型实现开门动画 |
| 想增加功能 | 添加计数 | 统计今天有多少人进门 |

**核心代码模式**
```
当绿旗被点击
将门的造型设为 [关闭]
重复执行
  如果 <红外传感器检测到障碍物?> 那么
    将门的造型设为 [打开]
    等待直到 <红外传感器未检测到障碍物?>
    等待 (2) 秒
    将门的造型设为 [关闭]
  结束
结束
```

**禁忌提醒**
❌ 不要把传感器对着强光源——会干扰检测
❌ 不要忘记延时关门——真实自动门都有��个功能

---

**核心知识点**
- 红外避障传感器：检测前方是否有障碍物
- 条件循环：持续检测并响应
- 延时控制：等待一段时间再执行

### 第4课：声控小夜灯

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-1-04 |
| **课程名称** | 声控小夜灯 |
| **认知负荷** | 中 |
| **核心技能** | 声音传感器、响度检测、阈值设定 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，晚上起床上厕所，要摸黑找开关是不是很麻烦？如果拍拍手灯就亮了，是不是很方便？今天我们就来做一个声控小���灯！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "声音传感器就像耳朵，能'听到'声音的大小。
>
> 声音的大小用'响度'来表示（0-100）：
> - 0 表示很安静
> - 100 表示非常吵
>
> 我们设置一个阈值，比如响度大于30就算'拍手'。
>
> 但有个问题：拍一次手，灯亮了；再拍一次，灯要灭。这需要用'状态切换'！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 环境太吵 | 提高阈值 | 从30调到50 |
| 检测不到 | 降低阈值 | 从30调到20 |
| 想增加功能 | 添加亮度调节 | 连续拍手调节亮度 |

**核心代码模式**
```
当绿旗被点击
将 [灯状态] 设为 [关]
重复执行
  如果 <(响度) > (30)> 那么
    如果 <(灯状态) = [关]> 那么
      将 [灯状态] 设为 [开]
      将灯的造型设为 [亮]
    否则
      将 [灯状态] 设为 [关]
      将灯的造型设为 [暗]
    结束
    等待 (0.5) 秒  // 防止连续触发
  结束
结束
```

---

**核心知识点**
- 声音传感器：检测环境声音响度（0-100）
- 阈值设定：确定触发条件
- 状态切换：用变量记录当前状态并切换

| **核心技能** | 倾斜传感器、方向检测、角色旋转 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们玩过不倒翁吗？推它一下，它会晃来晃去，但永远不会倒！今天我们用倾斜传感器来做一个电子不倒翁！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "倾斜传感器能感知设备的倾斜方向。
>
> 它会告诉我们：
> - 向左倾斜
> - 向右倾斜
> - 向前倾斜
> - 向后倾斜
> - 没有倾斜（水平）
>
> 我们让不倒翁跟着倾斜方向摇摆：向左倾斜，不倒翁就向左倒；向右倾斜，不倒翁就向右倒。但它永远会回到中间！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 反应太慢 | 减少等待时间 | 从0.1秒改为0.05秒 |
| 想增加趣味 | 添加表情 | 倾斜时显示惊讶表情 |
| 想增加难度 | 添加音效 | 不同方向播放不同声音 |

**核心代码模式**
```
当绿旗被点击
面向 (90) 度
重复执行
  如果 <倾斜传感器 = [向左]> 那么
    面向 (60) 度
  否则
    如果 <倾斜传感器 = [向右]> 那么
      面向 (120) 度
    否则
      面向 (90) 度
    结束
  结束
结束
```

---

**核心知识点**
- 倾斜传感器：检测设备倾斜方向
- 方向检测：左、右、前、后、水平
- 角色旋转：用"面向X度"控制角色朝向

> "小朋友们，邪恶的龙博士又来捣乱了！我们要用超声波武器打败他！超声波是什么？蝙蝠就是用超声波来'看'东西的！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "超声波传感器的工作原理：
> 1. 发出超声波（人耳听不到的声音）
> 2. 超声波碰到物体后反弹回来
> 3. 根据往返时间计算距离
>
> 就像你对着山喊一声，听到回声需要的时间越长，山就越远！
>
> 我们用超声波测量手的距离：
> - 手离得近（<10cm）→ 发射子弹
> - 手离得远 → 不发射
>
> 子弹用'克隆'技术：一个子弹角色可以克隆出很多个！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 测距不准 | 调整传感器角度 | 确保正对目标 |
| 子弹太多 | 添加冷却时间 | 每0.5秒才能发射一次 |
| 想增加难度 | 龙博士会移动 | 让目标随机移动 |

**核心代码模式**
```
// 发射子弹
当绿旗被点击
隐藏
重复执行
  如果 <(超声波距离) < (10)> 那么
    克隆 [自己]
    等待 (0.3) 秒
  结束
结束

// 子弹飞行
当作为克隆体启动时
移到 [玩家] 位置
显示
重复执行 (50) 次
  移动 (10) 步
  如果 <碰到 [龙博士] ?> 那么
    将 [得分] 增加 (1)
    删除此克隆体
  结束
结束
删除此克隆体
```

**禁忌提醒**
❌ 不要让传感器对着软布——超声波会被吸收
❌ 不要测量太远的距离——超过2米不准确

---

**核心知识点**
- 超声波测距：发射-反射-计算距离
- 克隆技术：一个角色复制多个实例
- 碰撞检测：检测子弹是否击中目标

**教师话术**
> "夏天好热啊！如果有一个风扇，天热了自动转快，天凉了自动转慢，是不是很棒？今天我们就来做一个智能风扇！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "智能风扇需要'感知'温度，所以我们用温度传感器！
>
> 温度传感器会返回当前温度（摄氏度）。
>
> 我们设计三档风速：
> - 温度 < 25°C → 风扇不转
> - 25°C ≤ 温度 < 30°C → 低速
> - 温度 ≥ 30°C → 高速
>
> 这就是'分段函数'的概念——不同范围有不同的处理方式！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 温度变化慢 | 用手捂传感器 | 体温可以让温度上升 |
| 想增加档位 | 添加中速档 | 四档或五档风速 |
| 想增加功能 | 添加显示 | 显示当前温度和档位 |

**核心代码模式**
```
当绿旗被点击
重复执行
  如果 <(温度) < (25)> 那么
    电机停止
    说 [风扇关闭]
  否则
    如果 <(温度) < (30)> 那么
      电机功率设为 (50)
      说 [低速运转]
    否则
      电机功率设为 (100)
      说 [高速运转]
    结束
  结束
  等待 (1) 秒
结束
```

---

**核心知识点**
- 温度传感器：测量环境温度
- 分段控制：不同温度范围对应不同动作
- 电机调速：通过功率控制转速

```
  否则
    如果 <(温度) < (20)> 那么
      说 [有点凉，穿外套吧！]
      将造型设为 [外套]
    否则
      如果 <(温度) < (28)> 那么
        说 [温度适中，穿长袖吧！]
        将造型设为 [长袖]
      否则
        说 [好热！穿短袖吧！]

    结束
  结束
  等待 (2) 秒
结束
```

**禁忌提醒**
❌ 不要把嵌套顺序搞反——要从小到大或从大到小判断

**核心知识点**
- 嵌套逻辑：如果-否则-如果的多层结构

| **认知负荷** | 高 |
| **核心技能** | RGB全彩灯、四位数码管、逻辑"与" |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "交通信号灯有三种颜色：红、黄、绿。我们用RGB灯来实现！
>
> RGB是什么？R=红(Red)，G=绿(Green)，B=蓝(Blue)。
> 每种颜色的亮度从0到255：
> - 红灯：R=255, G=0, B=0
> - 绿灯：R=0, G=255, B=0
> - 黄灯：R=255, G=255, B=0（红+绿=黄）
>
> 倒计时用四位数码管显示。
>
> 信号灯的逻辑：
> 1. 绿灯亮30秒
> 2. 黄灯亮3秒（警告）
> 3. 红灯亮30秒
> 4. 循环"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| RGB混淆 | 用调色板演示 | 展示颜色混合效果 |
| 想增加功能 | 添加行人灯 | 红灯时行人绿灯 |
| 想增加难度 | 添加紧急模式 | 按钮触发全红灯 |

**核心代码模式**
```
当绿旗被点击
重复执行
  // 绿灯30秒
  RGB灯设为 (0) (255) (0)
  重复 (30) 次
    数码管显示 (30 - 计数器)
    等待 (1) 秒
  结束

  // 黄灯3秒
  RGB灯设为 (255) (255) (0)
  重复 (3) 次
    数码管显示 (3 - 计数器)
    等待 (1) 秒
  结束

  // 红灯30秒
  RGB灯设为 (255) (0) (0)
  重复 (30) 次
    数码管显示 (30 - 计数器)
    等待 (1) 秒
  结束
结束
```

---

**核心知识点**
- RGB颜色模型：三原色混合（0-255）
- 四位数码管：显示数字
- 状态循环：绿→黄→红→绿...

### 第10课：公路大冒险

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，想不想开赛车？今天我们用电位器做一个方向盘，控制赛车在公路上飞驰！"

---

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "电位器是什么？就是一个可以旋转的旋钮！
>
> 旋转电位器，它会输出一个数字（0-1023）：
> - 转到最左边 → 0
> - 转到中间 → 512
> - 转到最右边 → 1023
>
> 我们把这个数字'映射'到赛车的位置：
> - 0 → 赛车在最左边
> - 1023 → 赛车在最右边
>
> 背景滚动的技巧：让背景图片不断向下移动，看起来就像赛车在前进！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 控制不灵敏 | 调整映射范围 | 缩小有效范围 |
| 想增加难度 | 添加障碍物 | 躲避路上的障碍 |
| 想增加功能 | 添加加速 | 按按钮加速 |

**核心代码模式**
```
当绿旗被点击
重复执行
  // 电位器控制赛车位置
  将 x 设为 (((电位器值) - 512) / 512 * 200)
  // 电位器512时x=0（中间），0时x=-200（左），1023时x=200（右）
结束

// 背景滚动（背景角色）
当绿旗被点击
重复执行
  将 y 增加 (-5)
  如果 <(y) < (-360)> 那么
    将 y 设为 (360)
  结束
结束
```

---

**核心知识点**
- 电位器：旋转式模拟量输入（0-1023）
- 数值映射：将一个范围的值转换到另一个范围
- 背景滚动：制造运动的视觉效果

**教师话术**
> "小朋友们，你们见过机器人的手吗？机器人的手指能弯曲、伸直，就像我们的手一样！今天我们来做一个机械手指！"

**步骤2：任务分析（20分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "机械手指的关键是'舵机'（也叫伺服电机）。
>
> 舵机和普通电机不一样：
> - 普通电机：只能转圈圈，不能停在固定位置
> - 舵机：可以精确转到指定角度（0°-180°）并保持
>

> - 电位器转到0 → 舵机转到0°（手指伸直）
> - 电位器转到1023 → 舵机转到180°（手指弯曲）
>

| 角度不准 | 校准舵机 | 调整初始位置 |

**核心代码模式**
```
当绿旗被点击
重复执行
  // 将电位器值(0-1023)映射到舵机角度(0-180)
  将 [角度] 设为 ((电位器值) / 1023 * 180)
  舵机转到 (角度) 度
  等待 (0.05) 秒
结束
```

---

**核心知识点**
- 舵机（伺服电机）：精确角度控制（0°-180°）
- 角度映射：电位器值→舵机角度
- 实时控制：持续读取输入并更新输出
