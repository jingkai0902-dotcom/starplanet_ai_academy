### 第1课：六足机器人

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-3-01 |
| **课程名称** | 六足机器人 |
| **认知负荷** | 高 |
| **核心技能** | 三角形稳定性、三角步态、差速转向 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们观察过昆虫是怎么走路的吗？蚂蚁、蜘蛛、螳螂...它们都有很多条腿。今天我们要制作一个六足机器人，像昆虫一样稳稳地走路！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "六足机器人为什么能稳定行走？秘密就在于'三角步态'！
>
> 想象一下：六条腿分成两组，每组三条腿。当一组腿抬起来迈步时，另一组三条腿稳稳地支撑在地上，形成一个三角形。三角形是最稳定的形状！
>
> 转向怎么实现呢？用'差速转向'——左边的腿走得慢，右边的腿走得快，机器人就会向左转。就像坦克一样！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 不理解三角步态 | 用手指演示 | 三根手指支撑，另三根移动 |
| 机器人不稳 | 检查重心 | 确保重心在支撑三角形内 |
| 想增加功能 | 添加避障 | 用超声波传感器检测障碍物 |

---

**核心知识点**
- 三角形稳定性：三点确定一个平面，最稳定
- 三角步态：六足分两组交替运动
- 差速转向：两侧速度差实现转向

### 第2课：投篮机

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-3-02 |
| **课程名称** | 投篮机 |
| **认知负荷** | 高 |
| **核心技能** | 曲柄滑块机构、红外传感器计数、去抖动(等待直到) |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们喜欢打篮球吗？今天我们要做一个自动投篮机，它能自动把球投进篮筐，还能自动计分！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "投篮机的核心是'曲柄滑块机构'。还记得我们学过的曲柄吗？曲柄转动时，连接的滑块会前后移动。我们用这个原理来推动篮球！
>
> 计分怎么实现呢？用红外传感器！当球经过篮筐时，会挡住红外线，传感器就知道进球了。
>
> 但有个问题：球经过时可能会被检测好几次。怎么办？用'等待直到'指令——检测到球后，等待球完全离开再继续检测。这叫'去抖动'！"

**核心代码模式**
```
当绿旗被点击
将 [得分] 设为 (0)
重复执行
  如果 <红外传感器检测到遮挡> 那么
    将 [得分] 增加 (1)
    等待直到 <红外传感器未检测到遮挡>  // 去抖动
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 重复计分 | 增加等待时间 | 或使用"等待直到"去抖动 |
| 投篮力度不够 | 调整曲柄长度 | 曲柄越长，推力越大 |
| 想增加难度 | 添加计时器 | 限时投篮挑战 |

---

**核心知识点**
- 曲柄滑块机构：圆周运动→直线往复运动
- 红外传感器计数：检测物体通过
- 去抖动(Debounce)：避免重复触发

### 第3课：自动花瓣机

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-3-03 |
| **课程名称** | 自动花瓣机 |
| **认知负荷** | 高 |
| **核心技能** | 齿轮传动比、双电机协作、繁花曲线原理 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过万花尺吗？用它可以画出漂亮的花瓣图案。今天我们要做一个自动花瓣机，让机器人帮我们画出美丽的繁花曲线！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "繁花曲线是怎么画出来的呢？秘密在于'齿轮传动比'！
>
> 我们用两个电机：一个电机让画笔绕着中心转，另一个电机让画笔自己也在转。当两个转速不一样时，画笔走过的轨迹就会形成花瓣图案！
>
> 传动比不同，花瓣的数量也不同。比如大齿轮40齿，小齿轮8齿，传动比是5:1，就会画出5瓣的花！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 图案不清晰 | 调整画笔压力 | 确保画笔接触纸面 |
| 想画不同花瓣 | 更换齿轮组合 | 改变传动比 |
| 理解困难 | 先手动演示 | 用万花尺体验 |

---

**核心知识点**
- 齿轮传动比：大齿轮齿数/小齿轮齿数
- 双电机协作：两个运动叠加产生复杂轨迹
- 繁花曲线(Spirograph)：数学与艺术的结合

### 第7课：坦克大战

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-3-07 |
| **课程名称** | 坦克大战 |
| **认知负荷** | 高 |
| **核心技能** | 履带传动、压强分布、红外通讯 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过坦克吗？坦克为什么能在泥地、沙地上行驶，而普通汽车却会陷进去呢？今天我们要制作两辆坦克，还能互相对战！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "坦克的秘密在于'履带'！履带把坦克的重量分散到很大的面积上，这样每个点受到的压力就小了。这就是'压强'的概念——同样的力，面积越大，压强越小！
>
> 履带传动的原理：电机带动驱动轮，驱动轮带动履带，履带再带动负重轮。整个履带就像一条移动的路面！
>
> 对战怎么实现呢？用红外通讯！一辆坦克发射红外信号，另一辆坦克接收到就算被击中。"

**核心代码模式**
```
// 发射端
当按下 [空格] 键
  红外发射 [攻击信号]
  播放声音 [发射]

// 接收端
当绿旗被点击
将 [生命值] 设为 (3)
重复执行
  如果 <红外接收到 [攻击信号]> 那么
    将 [生命值] 增加 (-1)
    播放声音 [被击中]
    如果 <(生命值) = (0)> 那么
      说 [游戏结束]
      停止 [全部]
    结束
  结束
结束
```

**变体示例**

### 第9课：巡线小车

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-3-09 |
| **课程名称** | 巡线小车 |
| **认知负荷** | 高 |
| **核心技能** | 红外反射原理、巡线算法、反馈控制 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过工厂里的自动搬运车吗？它们能沿着地上的线自动行驶，不需要人控制！今天我们要制作一辆巡线小车，让它自动沿着黑线走！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "巡线小车怎么'看见'黑线呢？用红外传感器！
>
> 红外传感器发出红外光，白色地面会反射回来，黑色线条会吸收掉。传感器检测反射光的强弱，就知道下面是白色还是黑色了！
>
> 巡线算法是这样的：
> - 左传感器检测到黑线 → 向左转
> - 右传感器检测到黑线 → 向右转
> - 两个都没检测到 → 直走
> - 两个都检测到 → 停止（到达终点）
>
> 这就是'反馈控制'——根据传感器的反馈不断调整方向！"

**核心代码模式**
```
当绿旗被点击
重复执行
  如果 <<左传感器检测到黑线> 且 <右传感器未检测到黑线>> 那么
    左转
  否则
    如果 <<右传感器检测到黑线> 且 <左传感器未检测到黑线>> 那么
      右转
    否则
      如果 <<左传感器未检测到黑线> 且 <右传感器未检测到黑线>> 那么
        直走
      否则
        停止  // 两个都检测到，到达终点
      结束
    结束
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 小车跑偏 | 调整传感器位置 | 两个传感器要对称 |
| 转弯太急 | 降低转向速度 | 或增加传感器间距 |
| 想增加功能 | 添加速度控制 | 直线加速，弯道减速 |

**禁忌提醒**
❌ 不要在强光下测试——会干扰红外传感器
❌ 不要用反光材料画线——红外光会被反射

---

**核心知识点**
- 红外反射原理：白色反射，黑色吸收
- 巡线算法：根据左右传感器状态决定转向
- 反馈控制：传感器→判断→执行→传感器（闭环）

```
将 [目标楼层] 设为 (2)
如果 <(当前楼层) < (目标楼层)> 那么
  电机正转  // 上升
  等待直到 <2楼传感器检测到电梯>
  电机停止
  将 [当前楼层] 设为 (2)
否则
  如果 <(当前楼层) > (目标楼层)> 那么
    电机反转  // 下降
    等待直到 <2楼传感器检测到电梯>
    电机停止
    将 [当前楼层] 设为 (2)
  结束
结束
```
