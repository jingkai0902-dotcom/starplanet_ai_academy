## 单元概览

| 项目 | 内容 |
|------|------|
| **单元编号** | 2-2 |
| **单元名称** | 智能交互系统 (Smart Interactive Systems) |
| **适用年龄** | 二年级（7-8岁） |
| **课时数** | 12节 × 90分钟 |
| **核心目标** | 掌握各类传感器与执行器的协同工作，理解控制闭环 |
| **主要教具** | 触碰开关、红外传感器、超声波传感器、声音传感器、舵机、蜂鸣器、RGB灯 |

**底层逻辑**
CODE2-2是该阶段的核心模块，重点让学生理解"输入（感知）- 处理（逻辑）- 输出（执行）"的计算机控制闭环。

**单元教学策略**
- 基础控制（01-04）：触碰、红外、数码管
- 进阶应用（05-08）：蜂鸣器、超声波、逻辑运算
- 综合系统（09-12）：语音合成、声音传感器、舵机

### 第2课：智能垃圾桶

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-2-02 |
| **课程名称** | 智能垃圾桶 |
| **认知负荷** | 中 |
| **核心技能** | 红外传感器、舵机控制、自动开合 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们扔垃圾的时候是不是要用手掀开盖子？手脏了怎么办？今天我们来做一个智能垃圾桶，手一靠近盖子就自动打开！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "智能垃圾桶需要两个部件：
> 1. **红外传感器**：检测有没有手靠近
> 2. **舵机**：控制盖子开合
>
> 工作流程：
> - 红外检测到手 → 舵机转到90度（盖子打开）
> - 红外没检测到 → 等待2秒 → 舵机转到0度（盖子关闭）
>
> 为什么要等2秒？给你时间把垃圾扔进去啊！"

**核心代码模式**
```
当绿旗被点击
舵机转到 (0) 度  // 初始关闭
重复执行
  如果 <红外传感器检测到?> 那么
    舵机转到 (90) 度  // 打开
    等待直到 <红外传感器未检测到?>
    等待 (2) 秒
    舵机转到 (0) 度   // 关闭
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 盖子开合太快 | 添加缓动 | 分多步转动，每步等待0.1秒 |
| 想增加功能 | 添加语音 | 打开时说"谢谢" |
| 红外不灵敏 | 调整位置 | 确保传感器朝向正确 |

---

**核心知识点**
- 红外传感器：检测物体靠近
- 舵机控制：精确角度定位
- 延时关闭：给用户操作时间

### 第3课：智能存钱罐

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-2-03 |
| **课程名称** | 智能存钱罐 |
| **认知负荷** | 中-高 |
| **核心技能** | 红外传感器、变量计数、四位数码管 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们有存钱罐吗？每次想知道存了多少钱，是不是要把钱倒出来数？今天我们做一个智能存钱罐，每投一枚硬币它就自动计数！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "智能存钱罐的原理和自动计票器很像：
> 1. 硬币投入时经过红外传感器
> 2. 传感器检测到遮挡，计数+1
> 3. 数码管显示当前总数
>
> 但有个问题：不同面值的硬币怎么办？
> - 简单版：只计数量，不管面值
> - 进阶版：用按钮选择面值（1元/5角/1角）"

**核心代码模式**
```
当绿旗被点击
将 [硬币数] 设为 (0)
将 [总金额] 设为 (0)
重复执行
  如果 <红外传感器检测到遮挡> 那么
    将 [硬币数] 增加 (1)
    将 [总金额] 增加 (1)  // 假设都是1元
    四位数码管显示 (总金额)
    播放声音 [叮]
    等待 (0.5) 秒  // 防抖
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 重复计数 | 增加等待时间 | 硬币通过需要时间 |
| 想区分面值 | 添加按钮 | 投币前按按钮选择面值 |
| 想增加功能 | 添加目标 | 设定存钱目标，达到后庆祝 |

---

**步骤3：家长班会（10分钟）** `#低负荷-热身` `#IFC-复盘`

**教师话术**
> "各位家长好！今天孩子们做了智能存钱罐。这个项目巩固了红外传感器和变量计数的知识，还培养了孩子的理财意识。回家后可以让孩子用这个存钱罐真正存钱！"

---

**核心知识点**
- 红外传感器：检测硬币通过
- 变量累加：记录硬币数量和金额
- 数码管显示：实时显示总金额
- 防抖处理：避免一枚硬币被计多次

### 第4课：自动计票器

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 2-2-04 |
| **课程名称** | 自动计票器 |
| **认知负荷** | 中-高 |
| **核心技能** | 红外传感器、四位数码管、变量累加 |

#### 详细教学流程

**步骤1：课堂问候（2分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们好！今天我们要做一个超酷的自动计票器！你们知道选举投票的时候，工作人员是怎么数票的吗？如果用手数，又慢又容易出错。今天我们就来做一个自动计票的机器！"

---

**步骤2：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "【播放动画】斯斯的班级要选班长，大家把票投进票箱。可是数票太麻烦了，斯斯想：能不能做一个机器，每投进一张票就自动加1呢？"

---

**步骤3：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "我们来分析一下自动计票器需要什么：
> 1. **感知**：怎么知道有票投进来？（红外传感器检测遮挡）
> 2. **记录**：怎么记住投了多少票？（变量！每次+1）
> 3. **显示**：怎么让大家看到票数？（四位数码管）
>
> 红外传感器的原理：发射红外光，如果有东西挡住，就检测到了。就像超市的自动门一样！"

**核心代码模式**
```
当绿旗被点击
将 [票数] 设为 (0)
重复执行
  如果 <红外传感器检测到遮挡> 那么
    将 [票数] 增加 (1)
    四位数码管显示 (票数)
    等待 (0.5) 秒  // 防止重复计数
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 重复计数 | 增加等待时间 | 票通过传感器需要时间 |
| 数码管不显示 | 检查接线 | VCC/GND/CLK/DIO四根线 |
| 想增加功能 | 添加声音提示 | 每计一票蜂鸣器响一声 |

**禁忌提醒**
❌ 不要让红外传感器对着强光——会干扰检测
❌ 不要忘记初始化变量为0——否则从上次的值开始

---

**步骤4：家长班会（10分钟）** `#低负荷-热身` `#IFC-复盘`

**教师话术**
> "各位家长大家好！今天孩子们学习了'自动计票器'。他们掌握了三个重要概念：
> 1. **红外传感器**：检测物体遮挡
> 2. **变量累加**：每次+1的计数逻辑
> 3. **四位数码管**：数字显示输出
>
> 这种'感知-计算-显示'的模式在生活中很常见，比如超市的客流统计、停车场的车位计数等。"

---

**核心知识点**
- 红外传感器：检测物体遮挡（有/无）
- 变量累加：`票数 = 票数 + 1`
- 四位数码管：显示0-9999的数字
- 防抖处理：等待一段时间避免重复计数

**底层逻辑**
- **事件驱动**：传感器触发→执行动作
- **状态记忆**：变量保存当前状态
- **人机交互**：数码管实时反馈
