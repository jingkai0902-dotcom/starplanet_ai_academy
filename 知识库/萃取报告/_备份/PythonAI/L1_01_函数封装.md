### 1-1-6 言听计从1

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 1-1-6 |
| **课程名称** | 言听计从1 |
| **认知负荷** | 高 |
| **核心技能** | 事件监听、pet.on()、hear/spawn事件 |

#### 详细教学流程

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天我们学习一个神奇的概念——**事件监听**！之前我们的程序是'顺序执行'，今天要转向'事件驱动'思维。"
>
> "**事件监听**就是让程序'等待'某件事发生，然后自动执行对应的函数：
> ```python
> def speak(event):
>     pet.say('喵~')
>
> pet.on('hear', speak)  # 当听到声音时，执行speak函数
> hero.say('你好小猫')    # 触发hear事件
> ```
>
> **事件类型**：
> - `'hear'`：听到说话时触发
> - `'spawn'`：物体/单位生成时触发"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 不理解事件驱动 | 用门铃类比 | "就像门铃，有人按了才响，不是一直响" |
| 忘记绑定事件 | 强调pet.on() | "必须先用on()绑定，事件才会触发函数" |
| 想监听更多事件 | 介绍spawn | "spawn事件在物品生成时触发" |

---

**步骤2：闯关实践（35分钟）** `#高负荷-产出` `#IFC-即时`

**教师话术**
> "进入关卡'边地好伙伴'！让你的宠物听到暗号后回应。然后挑战'森林加农炮'，结合循环与事件的复杂应用！"

**核心代码模式**

```python
# 示例1：听到声音后回答（Hear Event）
def speak(event):
    pet.say("喵~")

pet.on("hear", speak)
hero.say("你好小猫")

# 示例2：物品生成后去捡（Spawn Event）
def goFetch():
    potion = hero.findNearestItem()
    if potion:
        pet.fetch(potion)

pet.on("spawn", goFetch)
```

**禁忌提醒**
❌ 忘记绑定事件——函数定义了但不会被触发
❌ 事件类型拼写错误——'hear'不是'here'

---

**核心知识点**
- 事件监听：`pet.on("event_type", function_name)`
- 事件处理函数：当事件发生时自动调用的函数
- 事件类型：hear（听到说话）、spawn（物体生成）

**教学类比**
> 事件监听就像设置闹钟，闹钟响了（事件发生）才会叫你起床（执行函数）。

#### 课程基本信息

#### 详细教学流程

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天学习Python的**算术运算符**，有些你可能没见过！"
>
> "常见运算符：
> - `+` `-` `*` `/`：加减乘除
> - `//`：**整除**（只取整数部分）
> - `%`：**取余**（除法的余数）
> - `**`：**幂运算**（乘方）
>
> ```python
> print(7 // 2)   # 输出: 3（整除）

> print(2 ** 3)   # 输出: 8（2的3次方）
> ```
>
> 还有一个神奇的**eval()函数**，可以把字符串当代码执行：
> ```python
> result = eval('2 + 3 * 4')
> print(result)  # 输出: 14
> ```"

|----------|----------|------|
| 混淆/和// | 对比演示 | 7/2=3.5，7//2=3 |
| 不理解取余 | 用分糖果类比 | "7颗糖分给2人，每人3颗，剩1颗" |
| 对eval好奇 | 安全提醒 | eval很强大但要小心，不要执行不信任的输入 |

### 1-1-10 关系运算

#### 详细教学流程

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天学习**关系运算符**，也叫比较运算符。它们的结果是**布尔值**：True或False。"
>
> "比较运算符：
> - `==`：等于（注意是两个等号！）
> - `!=`：不等于
> - `>`、`<`：大于、小于
> - `>=`、`<=`：大于等于、小于等于
>
> ```python
> print(5 == 5)   # True
> print(5 != 3)   # True
> print(5 > 10)   # False
> ```
>
> 还有一个经典问题：**如何交换两个变量的值？**
> ```python
> a, b = 10, 20
> a, b = b, a  # Python的优雅写法
> print(a, b)  # 输出: 20 10
> ```"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 混淆=和== | 反复强调 | "一个等号是赋值，两个等号是比较" |
| 不理解布尔值 | 用判断题类比 | "就像判断题，对就是True，错就是False" |
| 变量交换不理解 | 用杯子类比 | 传统方法需要第三个杯子，Python可以直接交换 |

## 教学注意事项

| 类别 | 注意事项 |
|------|----------|
| **形参与实参** | 反复强调形参是"占位符"，实参是"具体值" |
| **作用域** | 函数内部定义的变量（局部）在外部无法访问 |
| **调试技巧** | 使用print()输出变量值或函数返回值 |
| **命名规范** | 函数名用小写字母和下划线，见名知意 |
