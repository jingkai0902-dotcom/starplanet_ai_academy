#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 1-0-1 |
| **课程名称** | 字如泉涌 |
| **认知负荷** | 中 |
| **核心技能** | 私有变量、广播机制、克隆技术 |
| **平台工具** | Scratch打字练习工具 |

#### 详细教学流程

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "同学们，在正式学Python之前，我们先用Scratch来练习打字！为什么呢？因为Python是用键盘敲代码的，打字速度很重要！"
>
> "今天我们要学三个Scratch的高级技巧：
>
> **第一，私有变量**：每个角色都有自己的'小秘密'。
> - 普通变量：所有角色都能看到、都能改
> - 私有变量：只有这个角色自己能用
>
> **第二，广播**：角色之间'打电话'通知对方。
> - 广播消息：发送一个信号
> - 当收到消息：接收信号后执行动作
>
> **第三，克隆**：复制出很多一样的角色。
> - 克隆自己：创建一个分身
> - 当作为克隆体启动时：分身出现后做什么"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 不理解私有变量 | 用"书包"类比 | "每个人的书包里装的东西不一样" |
| 广播概念模糊 | 用"对讲机"类比 | "广播就像对讲机，喊一声大家都能听到" |
| 克隆理解困难 | 用"复印机"类比 | "克隆就像复印，复印出来的和原件一样" |

---

**步骤2：闯关实践（35分钟）** `#高负荷-产出` `#IFC-即时`

**教师话术**
> "现在我们来做一个打字练习工具！
>
> 第一步：创建字母角色，用克隆让字母从天上掉下来
> 第二步：用私有变量记录每个字母是什么
> 第三步：按对键盘，广播消息让字母消失
>
> 看谁能在1分钟内打对最多字母！"

**即时反馈检查点**
- 检查学生是否正确设置私有变量
- 检查广播消息名称是否一致
- 检查克隆体是否正确删除

**核心代码模式（Scratch积木）**
```
当绿旗被点击
  重复执行
    等待 (随机取数 1 到 2) 秒
    克隆 [自己]

当作为克隆体启动时
  将 [我的字母] 设为 (字母表 的第 (随机取数 1 到 26) 项)
  移到 x: (随机取数 -200 到 200) y: 180
  重复执行直到 <碰到 [边缘]>
    将 y 坐标增加 -5
  删除此克隆体
```

>

---

**核心知识点**
- 私有变量：`仅适用于当前角色`的变量设置

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天我们要给打字游戏加一个专业的评分系统！
>
> **评分算法设计**：
> - 准确率 = 打对数量 ÷ 总按键数量 × 100%
> - 速度分 = 打对数量 ÷ 游戏时间 × 60（每分钟打字数）
> - 综合得分 = 准确率 × 0.6 + 速度分 × 0.4
>
> **数据统计**：
> - 总按键次数
> - 正确次数
> - 错误次数
> - 游戏时长
>
> **排行榜**：
> - 用列表存储历史最高分
> - 每次游戏结束比较是否破纪录"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 不理解百分比 | 用分数解释 | "10个里面对了8个，就是80%" |
| 算法太复杂 | 简化公式 | 先只用准确率，不加权重 |
| 想要更复杂 | 增加评级 | S/A/B/C/D五个等级 |

> 2. 计算准确率并显示
> 3. 计算每分钟打字数（WPM）
> 4. 游戏结束显示详细成绩单
>
> 进阶挑战：加一个排行榜，记录历史最高分！"

**即时反馈检查点**

### 1-0-5 知己知彼

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 1-0-5 |
| **课程名称** | 知己知彼 |
| **认知负荷** | 高 |
| **核心技能** | 注释、无限循环、缩进规则 |

#### 详细教学流程

**步骤1：知识讲解（15分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "今天我们学三个重要的知识点：
>
> **第一，注释**：用 `#` 开头的文字，电脑会跳过不执行。注释是写给人看的，帮助我们理解代码。
> ```python
> # 这是一条注释，电脑不会执行
> hero.moveRight()  # 这也是注释，写在代码后面
> ```
>
> **第二，无限循环**：`while True:` 表示一直重复执行。
> ```python
> while True:
>     hero.moveRight()  # 英雄会一直往右走
> ```
>

**核心代码模式**
```python
while True:
    # 找最近的敌人
    enemy = hero.findNearestEnemy()

    if enemy:
        # 计算与敌人的距离
        distance = hero.distanceTo(enemy)

        # 根据距离和血量决定策略
        if distance < 10:
            # 敌人很近
            if enemy.health < 30:
                hero.attack(enemy)  # 血少直接打
            else:
                hero.shield()       # 血多先防御
        elif distance < 30:
            # 敌人中等距离
            hero.attack(enemy)
        else:
            # 敌人较远，移动靠近
            hero.moveXY(enemy.pos.x, enemy.pos.y)
    else:
        # 没有敌人，回到保护位置
        hero.moveXY(40, 30)
```

**禁忌提醒**
❌ 比较时用`=`而不是`==`——`if health = 50`会报错
❌ 属性名拼写错误——`enemy.helth`会报AttributeError
❌ 忘记处理None情况——`enemy.health`在enemy为None时会报错
