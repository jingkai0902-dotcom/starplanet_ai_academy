## 一、体系综述与核心逻辑

### 1.1 课程定位

- **目标人群**：6-10岁（小学一至三年级）
- **课程名称**："少儿编程+"课程
- **核心特点**：强调软硬结合

### 1.2 核心理念

**"编程+"**：不仅仅是写代码，而是融合电子电路、机械结构、软件编程的跨学科整合。

### 1.3 量化指标

| 类别 | 数量 |
|------|------|
| Scratch编程知识 | 300+ |
| 机械原理 | 50+ |
| 跨学科知识 | 120+ |
| 硬件搭建作品 | 80+ |
| 程序作品 | 150+ |

### 1.4 认知模型

遵循**"点-线-面-体"**的PBL认知发展模型：
```
点（数据）→ 线（算法）→ 面（抽象）→ 体（系统）
```

## 二、分级课程详情

### 2.1 CODE 1：编程启蒙与机械基础

**适用对象**：一年级（6-7岁），处于逻辑思维快速发展期

**核心教具**：LEGO WeDo 2.0套装 + iPad/电脑模块化编程

#### 四个子阶段

| 阶段 | 名称 | 核心内容 | 典型案例 |
|------|------|----------|----------|
| 1-1 | 编程模块小百科 | 皮带/齿轮传动、杠杆、马达控制 | 《跳舞小鸟》、《超智能赛车》 |
| 1-2 | 编程模块初应用 | 棘轮棘爪、蜗轮箱、传感器（运动/倾斜） | 《鳄鱼》、《智能电动门》 |

| 1-4 | 编程思维初建立 | 变量、广播、克隆 | 《斯斯摘苹果》、《单词接龙》 |

**能力目标**：
- 分解能力（拆分步���）

**对应出口**：

### 2.2 CODE 2：传感器应用与数理逻辑

**适用对象**：二年级（7-9岁），处于逻辑能力强化期

**核心教具**：OnlyScratch硬件套装、CircleBit创客开发板（集成光敏、声敏、陀螺仪等传感器）

#### 四个子阶段

|------|------|----------|----------|

**能力目标**：
- 抽象能力（物理信号转代码）
- 算法思维

**对应出口**：
- 机器人二级等级考
- 软编Scratch二级
- Robotex世界机器人大会

## 三、教学方法论

### 3.1 PBL项目制学习

- 不只是为了考级，而是通过解决问题驱动学习
- 例如："如何设计一个防近视台灯"
- 强调**"做中学"**，从现象出发，建立底层逻辑

### 3.2 5E探究式教学法

| 步骤 | 英文 | 内容 |
|------|------|------|
| 1 | Engage | 动画引入（机器人小白IP） |
| 2 | Explore | 动手搭建与实验 |
| 3 | Explain | 归纳原理（如齿轮比、循环语句） |
| 4 | Elaborate | 举一反三，解决新问题 |
| 5 | Evaluate | 作品展示与反思 |

### 3.3 八大能力模型

将孩子表现归纳为：
1. 观察力
2. 分析力
3. 理解力
4. 想象力
5. 判断力
6. 推理力
7. 动手能力
8. 语言能力

## 四、销售与规划策略

### 4.1 话术公式

```
现象(Phenomenon) → 做法(Method) → 能力(Ability) → 好处(Benefit)
```

**示例**：
- 现象：孩子写字慢
- 做法：练编程操作鼠标键盘
- 能力：提升手眼协调
- 好处：卷面分提高

### 4.2 规划逻辑

#### 低龄（1-2年级）
- 走CODE课程，打好Scratch基础
- 避免过早学Python（容易放弃）

#### 升学路径

**体制内路径**：
```
Scratch考级 → Python/C++ → 信奥赛(CSP-J/S) → 科技特长生
```

**体制外/综合路径**：
```
机器人考级 → VEX/FLL赛事 → 背景提升
```

### 4.3 紧迫感制造

- 利用"考级是白名单赛事的敲门砖"这一信息差
- 计算"剩余课时缺口"来促进续费

### 4.4 双版本规划

| 版本 | 特点 | 引导方向 |
|------|------|----------|
| 考级版 | 高频高压 | 高价值选择 |
| 兴趣版 | 寓教于乐 | 备选方案 |
