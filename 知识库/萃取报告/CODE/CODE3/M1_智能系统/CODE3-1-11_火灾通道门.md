## 详细教学流程

### 步骤1：情境引入（5分钟）
`#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们知道消防通道吗？平时消防门是关着的，但发生火灾时必须自动打开，让人们逃生！今天我们来做一个智能消防通道门！"

---

### 步骤2：任务分析（15分钟）
`#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "消防通道门的安全逻辑非常重要！
>
> 正常情况：
> - 门保持关闭
> - 刷卡才能开门
>
> 紧急情况（火灾）：
> - 温度超过60°C → 自动开门
> - 烟雾检测到 → 自动开门
> - **安全优先**：紧急情况下，不需要刷卡！
>
> 这就是'安全逻辑优先权'——安全永远是第一位的！
>
> 流程图：
> ```
> 开始 → 检测温度 → 温度>60°C?
>                    ↓是
>                  立即开门（紧急）
>                    ↓否
>                  检测刷卡 → 刷卡?
>                              ↓是    ↓否
>                            开门 → 保持关闭
> ```"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 温度不够高 | 用手捂传感器 | 体温可以触发测试 |
| 想增加功能 | 添加报警 | 紧急开门时同时报警 |
| 想增加功能 | 添加广播 | 播放疏散语音 |

**核心代码模式**
```
当绿旗被点击
将 [紧急温度] 设为 (60)
舵机转到 (0) 度  // 门关闭

重复执行
  // 安全优先：先检测紧急情况
  如果 <(温度) > (紧急温度)> 那么
    // 紧急情况：立即开门
    舵机转到 (90) 度
    RGB灯设为 [红色]
    蜂鸣器响
    说 [紧急！火灾警报！请立即疏散！]
  否则
    // 正常情况：需要刷卡
    如果 <刷卡检测到?> 那么
      舵机转到 (90) 度  // 开门
      等待 (5) 秒
      舵机转到 (0) 度   // 关门
    结束
  结束
结束
```

**禁忌提醒**
- 不要把紧急开门条件设得太高——安全第一
- 不要让正常开门覆盖紧急开门——紧急情况优先级最高

### 步骤3：家长班会（10分钟）
`#低负荷-热身` `#IFC-复盘`

**教师话术**
> "各位家长好！今天孩子们学习了'火灾通道门'。这节课的核心是'安全逻辑优先权'：
> 1. **安全第一**：紧急情况下，安全逻辑优先于其他逻辑
> 2. **流程图设计**：用流程图理清复杂逻辑
> 3. **温度传感器**：检测环境温度
>
> 这是一个非常有教育意义的项目，让孩子们理解安全设计的重要性！"

---

## 核心知识点

- **安全逻辑优先权**：紧急情况优先于正常逻辑
- **流程图应用**：设计复杂逻辑流程
- **温度传感器**：检测环境温度变化

**底层逻辑**
- **安全设计**：安全永远是第一位
- **优先级**：不同条件有不同优先级
- **冗余设计**：多重检测确保安全
