## 详细教学流程

### 步骤1：情境引入（5分钟）

`#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过坦克吗？坦克为什么能在泥地、沙地上行驶，而普通汽车却会陷进去呢？今天我们要制作两辆坦克，还能互相对战！"

---

### 步骤2：任务分析（15分钟）

`#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "坦克的秘密在于'履带'！履带把坦克的重量分散到很大的面积上，这样每个点受到的压力就小了。这就是'压强'的概念——同样的力，面积越大，压强越小！
>
> 履带传动的原理：电机带动驱动轮，驱动轮带动履带，履带再带动负重轮。整个履带就像一条移动的路面！
>
> 对战怎么实现呢？用红外通讯！一辆坦克发射红外信号，另一辆坦克接收到就算被击中。"

---

### 核心代码模式

```
// 发射端
当按下 [空格] 键
  红外发射 [攻击信号]
  播放声音 [发射]

// 接收端
当绿旗被点击
将 [生命值] 设为 (3)
重复执行
  如果 <红外接收到 [攻击信号]> 那么
    将 [生命值] 增加 (-1)
    播放声音 [被击中]
    如果 <(生命值) = (0)> 那么
      说 [游戏结束]
      停止 [全部]
    结束
  结束
结束
```

### 变体示例

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 履带脱落 | 调整张紧度 | 履带不能太松也不能太紧 |
| 红外通讯不稳定 | 调整角度 | 确保发射和接收对准 |
| 想增加功能 | 添加护盾 | 被击中后短暂无敌 |

## 核心知识点

- **履带传动**：将重量分散到大面积
- **压强**：P = F/S，面积越大压强越小
- **红外通讯**：无线信号传输
