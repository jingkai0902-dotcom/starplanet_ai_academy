---

### 步骤2：任务分析（15分钟）

`#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "大招太强了，如果随时都能用，游戏就太简单了。所以我们要加入'冷却时间'！
>
> 冷却时间的逻辑：
> 1. 按Q键释放大招
> 2. 大招释放后，开始10秒倒计时
> 3. 倒计时结束前，不能再次释放
> 4. 倒计时结束，大招可用
>
> 这就是游戏设计中的'平衡性'——让游戏不会太简单也不会太难！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 冷却太长 | 减少冷却时间 | 从10秒改为5秒 |
| 想要多种大招 | 添加不同技能 | 护盾、加速等 |
| UI不清晰 | 添加进度条 | 显示冷却剩余时间 |

**核心代码模式**
```
// 大招系统
当绿旗被点击
将 [冷却时间] 设为 (0)
将 [大招可用] 设为 [是]

// 冷却倒计时
当绿旗被点击
重复执行
  如果 <(冷却时间) > (0)> 那么
    将 [大招可用] 设为 [否]
    等待 (1) 秒
    将 [冷却时间] 增加 (-1)
  否则
    将 [大招可用] 设为 [是]
  结束
结束

// 释放大招
当按下 [q] 键
如果 <(大招可用) = [是]> 那么
  广播 [大招发动]
  播放声音 [爆炸]
  将 [冷却时间] 设为 (10)
结束

// 敌机响应大招
当收到 [大招发动]
删除此克隆体
```

## 核心知识点

- 冷却时间(Cooldown)：技能使用后的等待时间
- 游戏平衡：通过数值调整控制游戏难度
- UI设计：得分、冷却条、开始/结束界面
- 广播机制：一对多的消息传递

---

## 底层逻辑

- **游戏设计**：平衡性是游戏可玩性的关键
- **用户体验**：清晰的UI让玩家更容易上手
