#### 详细教学流程

**步骤1：课堂问候（2分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们好！欢迎来到CODE3的第一课！从今天开始，我们要学习更复杂的智能系统。你们知道吗？冰箱门如果忘记关，食物会坏掉，还会浪费电！今天我们来做一个冰箱报警器！"

---

**步骤2：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "【播放动画】斯斯的妈妈总是忘记关冰箱门，结果冰箱里的冰淇淋都化了！斯斯想：能不能做一个装置，冰箱门开太久就自动报警呢？"

---

**步骤3：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "冰箱报警器需要解决两个问题：
> 1. 怎么知道冰箱门开了？——用光线传感器！
> 2. 怎么知道开了'太久'？——用计时器！
>
> 光线传感器的原理：
> - 冰箱门关着 → 里面很暗 → 光线值很低（比如<100）
> - 冰箱门开了 → 光线照进来 → 光线值很高（比如>500）
>
> 这里用到了'逻辑非'：
> - 不是'光线亮就报警'
> - 而是'光线亮**超过30秒**才报警'
>
> 所以条件是：NOT(光线暗) AND (时间>30秒)"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 光线值不稳定 | 调整阈值 | 根据实际环境调整判断值 |
| 想增加功能 | 添加语音提示 | 报警时说"请关冰箱门" |
| 理解快 | 添加温度检测 | 结合温度传感器判断 |

**核心代码模式**
```
当绿旗被点击
将 [开门时间] 设为 (0)
重复执行
  如果 <(光线传感器) > (500)> 那么
    // 门开着，开始计时
    将 [开门时间] 增加 (1)
    等待 (1) 秒
    如果 <(开门时间) > (30)> 那么
      蜂鸣器响
      RGB灯设为红色
    结束
  否则
    // 门关了，重置计时
    将 [开门时间] 设为 (0)
    蜂鸣器停
    RGB灯设为绿色
  结束
结束
```

**禁忌提醒**
❌ 不要把光线传感器放在冰箱外面——检测不到门的状态
❌ 不要忘记重置计时器——否则关门后再开门会立即报警

---

**步骤4：家长班会（10分钟）** `#低负荷-热身` `#IFC-复盘`

**教师话术**

> 1. **光线传感器**：检测环境亮度
> 2. **计时器逻辑**：累计时间判断
> 3. **逻辑非(NOT)**：条件取反
>
> 这个项目结合了传感器和时间逻辑，是智能家居的入门应用！"

**核心知识点**
- 光线传感器：检测环境光强度（0-1023）
- 计时器：累计时间并判断
- 逻辑非(NOT)：条件取反，暗→亮的状态变化

### 第2课：神奇遥控器

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-1-02 |
| **课程名称** | 神奇遥控器 |
| **认知负荷** | 高 |
| **核心技能** | 红外通讯原理、广播机制、无线控制 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们家的电视遥控器是怎么工作的？为什么按一下按钮，电视就能收到信号呢？今天我们来揭秘遥控器的原理，还要自己做一个！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "遥控器的秘密是'红外通讯'！
>
> 红外通讯的原理：
> 1. 遥控器发出红外光（人眼看不见）
> 2. 红外光携带'编码'信息（比如'开机'='1001'）
> 3. 接收器收到信号，解码后执行命令
>
> 我们用两个设备：
> - **发射端**：按按钮 → 发送红外信号
> - **接收端**：收到信号 → 执行动作
>
> 在Scratch里，我们用'广播'来模拟这个过程：
> - 发射端：广播消息'开灯'
> - 接收端：当收到'开灯'消息时，执行开灯动作"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 信号收不到 | 检查对准 | 红外需要对准，不能有遮挡 |
| 想增加功能 | 多按钮控制 | 不同按钮发送不同命令 |
| 理解快 | 添加反馈 | 接收端收到后闪灯确认 |

**核心代码模式**
```
// 发射端代码
当按下 [按钮1] 时
红外发送 [1]  // 发送编码1

当按下 [按钮2] 时
红外发送 [2]  // 发送编码2

// 接收端代码
当绿旗被点击
重复执行
  如果 <(红外接收) = [1]> 那么
    RGB灯设为 [红色]
  结束
  如果 <(红外接收) = [2]> 那么
    RGB灯设为 [蓝色]
  结束
结束
```

**禁忌提醒**
❌ 不要让红外发射器和接收器距离太远——信号会衰减
❌ 不要在强光下测试——阳光会干扰红外信号

---

**核心知识点**
- 红外通讯：无线数据传输方式
- 编码/解码：信息的发送和接收
- 广播机制：一对多的消息传递

### 第3课：魔力电吉他

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-1-03 |
| **课程名称** | 魔力电吉他 |
| **认知负荷** | 高 |
| **核心技能** | 超声波测距、数学运算(线性变换)、音符控制 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过电吉他吗？今天我们要做一个更神奇的乐器——不用弦，用手在空中挥动就能演奏！就像魔法一样！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "魔力电吉他的原理：
> 1. 用超声波传感器测量手的距离
> 2. 把距离'转换'成音符
>
> 距离和音符的对应关系：
> - 手离得近（5cm）→ 低音Do
> - 手离得远（30cm）→ 高音Si
>
> 这里用到了'线性变换'（数学运算）：
> - 距离范围：5cm ~ 30cm（共25cm）
> - 音符范围：60 ~ 72（共12个音）
>
> 公式：音符 = 60 + (距离-5) / 25 × 12
>
> 简化版：音符 = 60 + (距离-5) × 0.48"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 音符跳动 | 添加平滑 | 取多次测量的平均值 |
| 想增加功能 | 添加节奏 | 用另一只手控制节拍 |
| 数学困难 | 简化映射 | 分成7个固定区间对应7个音 |

**核心代码模式**
```
当绿旗被点击
重复执行
  将 [距离] 设为 (超声波距离)
  // 限制距离范围
  如果 <(距离) < (5)> 那么
    将 [距离] 设为 (5)
  结束
  如果 <(距离) > (30)> 那么
    将 [距离] 设为 (30)
  结束
  // 线性变换：距离→音符
  将 [音符] 设为 (60 + ((距离) - 5) * 0.48)
  播放音符 (取整 (音符)) 持续 (0.2) 拍
结束
```

---

**核心知识点**
- 超声波测距：精确测量距离
- 线性变换：将一个范围的值映射到另一个范围
- 音符控制：MIDI音符编号（60=中央C）

#### 详细教学流程

**教师话术**
> "小朋友们，你们家楼道的灯是怎么控制的？有的楼道灯白天也亮着，很浪费电！今天我们来做一个智能楼梯灯，只有天黑**而且**有人的时候才亮！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "智能楼梯灯需要满足两个条件：
> 1. 天黑了（光线传感器值<200）
> 2. 有人经过（红外传感器检测到）
>
> 两个条件**都满足**才开灯，这就是'逻辑与(AND)'！
>
> 我们先画流程图：
> ```
> 开始 → 检测光线 → 光线暗?
>                    ↓是        ↓否
>              检测红外 → 不开灯
>                    ↓
>              有人?
>              ↓是    ↓否
>            开灯 → 不开灯
> ```
>
> 流程图帮助我们理清逻辑，再写代码就简单了！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 逻辑混乱 | 用真值表 | 画出所有情况的结果 |
| 想增加功能 | 添加延时 | 人走后灯延时30秒关闭 |
| 理解快 | 添加亮度调节 | 根据环境光调整灯的亮度 |

**核心代码模式**
```
当绿旗被点击
重复执行
  // 逻辑与：两个条件都满足才开灯
  如果 <<(光线传感器) < (200)> 且 <红外传感器检测到?>> 那么
    RGB灯设为 [白色]
    等待直到 <红外传感器未检测到?>
    等待 (30) 秒  // 延时关灯
    RGB灯设为 [关闭]
  结束
结束
```

---

**步骤3：流程图设计（10分钟）** `#中负荷-操练` `#IFC-即时`

**教师话术**
> "在写代码之前，我们先画流程图！
>
> 流程图的基本符号：
> - 椭圆：开始/结束
> - 矩形：处理步骤
> - 菱形：判断条件
> - 箭头：流程方向
>
> 画流程图的好处：
> 1. 理清思路，不会遗漏
> 2. 发现逻辑错误
> 3. 方便和别人交流"

---

**核心知识点**
- 逻辑与(AND)：两个条件都为真，结果才为真
- 流程图设计：用图形表示程序逻辑

### 第5课：家园卫士

#### 课程基本信息

| 项目 | 内容 |
|------|------|
| **课程编号** | 3-1-05 |
| **课程名称** | 家园卫士 |
| **认知负荷** | 高 |
| **核心技能** | 雷达扫描逻辑、舵机与传感器配合、安防系统 |

#### 详细教学流程

**步骤1：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，你们见过雷达吗？雷达可以360度扫描，发现任何靠近的物体！今天我们来做一个'家园卫士'，它能像雷达一样扫描周围，发现入侵者就报警！"

---

**步骤2：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "家园卫士的工作原理：
> 1. 舵机带着超声波传感器转动（0°→180°→0°）
> 2. 每转一个角度，测量一次距离
> 3. 如果距离小于警戒值，说明有入侵者！
>
> 雷达扫描的逻辑：
> ```
> 角度从0到180，每次增加10度
>   舵机转到当前角度
>   测量距离
>   如果距离<50cm，报警！
> 角度从180到0，每次减少10度
>   ...同上
> ```
>
> 这就是'扫描'——系统地检查每个方向！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 扫描太慢 | 增大步进角度 | 从10度改为20度 |
| 想增加功能 | 添加追踪 | 发现入侵者后持续对准 |
| 想可视化 | 屏幕显示 | 在Scratch舞台上画雷达图 |

**核心代码模式**
```
当绿旗被点击
将 [警戒距离] 设为 (50)
重复执行
  // 正向扫描：0→180
  将 [角度] 设为 (0)
  重复 (18) 次
    舵机转到 (角度) 度

      舵机转到 (0) 度   // 关门
    结束
  结束
结束
```
