**步骤2：情境引入（5分钟）** `#低负荷-热身` `#IFC-预防`

**教师话术**
> "【播放动画】斯斯的教室里总是很吵，老师说话都听不清。斯斯想：能不能做一个装置，当声音超过一定分贝就自动亮红灯提醒大家呢？"

---

**步骤3：任务分析（15分钟）** `#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "声音传感器可以检测环境的声音大小，返回一个数值（0-1023）。
>
> 我们的报警逻辑是：
> - 声音 < 300：安静，绿灯
> - 声音 >= 300 且 < 600：有点吵，黄灯
> - 声音 >= 600：太吵了，红灯+蜂鸣器
>
> 注意这里用到了'逻辑与'：声音 >= 300 **且** 声音 < 600，两个条件都满足才亮黄灯！"

**核心代码模式**
```
当绿旗被点击
重复执行
  如果 <(声音传感器) < (300)> 那么
    RGB灯设为 [绿色]
  否则
    如果 <<(声音传感器) >= (300)> 且 <(声音传感器) < (600)>> 那么
      RGB灯设为 [黄色]
    否则
      RGB灯设为 [红色]
      蜂鸣器响 (0.5) 秒
    结束
  结束
结束
```

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 阈值不合适 | 实测调整 | 让学生在不同环境测试 |
| 不理解逻辑与 | 用门禁类比 | "需要刷卡且输密码才能进" |
| 想增加功能 | 添加分贝显示 | 用数码管显示当前分贝值 |

**禁忌提醒**
- 不要把传感器放在蜂鸣器旁边——会形成反馈循环
- 不要在嘈杂环境调试——影响阈值判断

## 核心知识点

- 声音传感器：检测声音强度（0-1023）
- 逻辑与(AND)：`条件1 且 条件2`
- 多级阈值判断：if-else嵌套
- 反馈控制：根据输入调整输出

**底层逻辑**
- **模拟量处理**：声音是连续变化的数值
- **区间判断**：将连续值划分为离散区间
- **布尔代数**：逻辑与运算的实际应用
