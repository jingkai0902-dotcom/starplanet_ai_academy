## 详细教学流程

### 步骤1：情境引入（5分钟）
`#低负荷-热身` `#IFC-预防`

**教师话术**
> "小朋友们，早上出门不知道穿什么衣服怎么办？今天我们做一个穿衣提示仪，它会根据温度告诉你该穿什么！"

---

### 步骤2：任务分析（20分钟）
`#中负荷-操练` `#IFC-即时` ⭐核心环节

**教师话术**
> "穿衣提示仪的逻辑比智能风扇更复杂！
>
> 我们要判断多个温度区间：
> - 温度 < 10°C → 穿羽绒服
> - 10°C ≤ 温度 < 20°C → 穿外套
> - 20°C ≤ 温度 < 28°C → 穿长袖
> - 温度 ≥ 28°C → 穿短袖
>
> 这需要用'嵌套的如果'——一个如果里面套另一个如果！"

**变体示例**

| 学生情况 | 调整方案 | 说明 |
|----------|----------|------|
| 嵌套混乱 | 画流程图 | 先画清楚判断顺序 |
| 想增加功能 | 添加天气 | 结合湿度给出建议 |
| 想个性化 | 自定义温度 | 让用户设置自己的舒适温度 |

---

### 步骤3：编程设计（25分钟）
`#高负荷-产出` `#IFC-即时`

**核心代码模式**
```
当绿旗被点击
重复执行
  如果 <(温度) < (10)> 那么
    说 [好冷！穿羽绒服吧！]
    将造型设为 [羽绒服]
  否则
    如果 <(温度) < (20)> 那么
      说 [有点凉，穿外套吧！]
      将造型设为 [外套]
    否则
      如果 <(温度) < (28)> 那么
        说 [温度适中，穿长袖吧！]
        将造型设为 [长袖]
      否则
        说 [好热！穿短袖吧！]
        将造型设为 [短袖]
      结束
    结束
  结束
  等待 (2) 秒
结束
```

**禁忌提醒**
- 不要把嵌套顺序搞反——要从小到大或从大到小判断
- 不要忘记"否则"——每个条件都要有对应处理

---

## 核心知识点

- **嵌套逻辑**：如果-否则-如果的多层结构
- **多条件判断**：多个温度区间的分类
- **数值比较**：< ≤ > ≥ 的使用
