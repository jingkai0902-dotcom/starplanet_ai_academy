## 核心价值观

| 原则 | 说明 |
|------|------|
| **不删减逻辑** | 保留完整的业务逻辑链条，禁止为简洁而牺牲逻辑完整性 |

## 一、项目总体目标

### 1.1 知识库定位

构建一套**完整、专业、可落地**的斯坦星球运营知识体系，服务于：
- 日常工作查阅
- 培训体系搭建
- 考核系统开发
- 后续程序/系统开发的数据基础

## 三、内容萃取标准（原子化萃取标准）

> **原子化萃取原则**：每一条萃取内容都是一块独立的"知识砖块"，可被后续程序自动调用、组合、生成。

### 3.1 话术类内容标准

**必须包含的七要素：**

| 要素 | 说明 |
|------|------|
| **场景定义** | 谁、什么情况、什么目的 |
| **标准话术** | 可直接开口说的完整话术 |
| **底层逻辑** | 为什么这样说有效（心理学/销售学依据） |
| **变体示例** | 2-3个变体，适应不同情境 |
| **禁忌提醒** | 不能说什么、会导致什么后果 |
| **生命周期阶段** | #新招期/#体验期/#在读期/#续费期/#升班期/#流失预警期/#转介绍期 |
| **竞品对标** | 对标竞品、差异化点、数据支撑、配套道具 |

**话术萃取格式示例：**

```markdown
#### 场景：家长咨询价格时的价值锚定

**标准话术**
> "您问价格是对的，但我想先帮您算一笔账。咱们的课程是按课时收费的，每课时XX元。但关键是，孩子学完CODE 1阶段后，能独立完成一个完整的游戏项目，这个能力在小升初科技特长生面试中是加分项。您觉得这个投入值不值？"

**底层逻辑**
价值锚定效应：先建立价值认知，再谈价格。将抽象的"课时费"转化为具体的"升学竞争力"，让家长从"花钱"思维转向"投资"思维。

**变体示例**
1. 针对低龄家长："您看，孩子现在4岁，到小学三年级还有5年时间。如果现在开始培养编程思维，到时候参加CSP-J考试就是水到渠成的事。"
2. 针对高年级家长："您孩子现在五年级，正好是黄金窗口期。再晚一年，初中学业压力大，就很难系统学习了。"

**禁忌提醒**
❌ 不要说"我们比XX机构便宜"——这会把自己拉到价格战的泥潭
❌ 不要说"学编程以后好找工作"——家长关心的是眼前的升学，不是十几年后的就业

`#执行层` `#测评项` `#价值锚定` `#新招期`
[UID: SL-TC-001]
```

### 3.2 流程类内容标准

**必须包含的五要素：**

| 要素 | 说明 |
|------|------|
| **触发条件** | 什么情况下执行此流程 |
| **动作序列** | 每个步骤的具体动作 |
| **配套话术** | 该步骤需要说什么 |
| **检查清单** | 如何判断步骤完成 |
| **异常处理** | 如果出现问题怎么办 |

**流程萃取格式示例：**

```markdown
#### 流程：新学员首课接待SOP

**触发条件**
新学员第一次到校上课

**动作序列**

| 步骤 | 时间 | 动作 | 配套话术 | 检查清单 |
|------|------|------|----------|----------|
| 1 | 课前15分钟 | 前台确认学员信息 | "您好，请问是XX小朋友吗？我是今天的课程顾问小王。" | □ 核对姓名 □ 核对课程 |
| 2 | 课前10分钟 | 带领参观教室 | "这是我们的编程教室，每个孩子都有独立的电脑和教具。" | □ 介绍环境 □ 介绍老师 |
| 3 | 课前5分钟 | 交接给授课老师 | "这是XX老师，今天由她来带XX小朋友上课。" | □ 完成交接 □ 家长安顿 |
| 4 | 课中 | 家长等候区服务 | "您可以在这里休息，有茶水和WiFi，课程结束我会来通知您。" | □ 提供饮品 □ 告知时长 |
| 5 | 课后 | 反馈沟通 | "今天XX表现很棒，特别是在XX环节..." | □ 正面反馈 □ 预约下次 |

**异常处理**
- 学员迟到：电话确认，安排补课或顺延
- 学员哭闹：安抚情绪，必要时家长陪同
- 设备故障：立即更换备用设备，不影响课程进度

`#执行层` `#测评项`
[UID: SOP-FC-001]
```

| 规范 | 说明 | 示例 |
|------|------|------|

**常用教育学依据参考：**

| 理论/方法 | 适用场景 |
|-----------|----------|

| 同伴学习理论(Peer Learning) | 小组合作、代码互评 |
| 具身认知(Embodied Cognition) | 实物教具、动手操作 |

#### 3.6.4 教学内容与话术内容的区别

| 维度 | 教学内容 | 话术内容 |
|------|---------|---------|

| **心理学标签** | 不强制要求 | 必须标注 |
| **生命周期标签** | 不适用 | 必须标注 |
| **变体示例** | 按需提供 | 必须提供2-3个 |
| **UID格式** | `阶段-单元-序号` | `部门-类型-序号` |

## 六、萃取工作流程

### 6.1 快速扫描阶段

1. 浏览笔记本全部来源
2. 识别内容类型和价值密度
3. 生成《快速扫描报告》
4. 规划深度萃取优先级

### 6.2 深度萃取阶段

1. 按5-5循环原则分批处理
2. 执行多维扫描（话术/流程/理论/案例/规范）
3. 打标签、生成UID
4. 输出结构化萃取报告

### 6.3 全局合成阶段

1. 汇总所有萃取报告
2. 按用户层级重组内容
3. 生成最终知识库文档
4. 建立知识引用追溯索引

## 七、执行协议

### 7.1 分批萃取协议（5-5循环原则）

- 每次仅加载**5个**来源内容
- 完成一批后再处理下一批
- 禁止全量重读，从断点继续

**断点记录格式**：
```
断点位置：来源 1-5 / 共 48 个 | 最后Source ID: STEM-05
```

### 7.2 超级萃取提问模板

```
当前处理：[笔记本名称] 来源序号 [X-Y]。

请深度分析这5个来源，执行以下多维任务：

**第一步：多维扫描**
1. 话术逻���：场景定义、标准话术、底层逻辑、变体示例、禁忌提醒
2. 动作逻辑：触发条件、动作序列、配套话术、检查清单、异常处理
3. 底层逻辑：完整框架、核心概念、可执行的逻辑模型
4. 规范逻辑：操作边界、执行细节、奖惩逻辑
5. 教学内容：知识点、项目、教学流程、教学技巧

**第二步：结构化输出**
- 按标准模板输出
- 打多态标签
- 生成UID
- 标注底层逻辑和禁忌提醒

**第三步：空白识别**
- 识别知识缺口
- 生成补漏任务
```

### 7.3 动态补漏指令

当识别到知识缺口时，记录以下信息：
- 缺口类型（话术/流程/案例/规范）
- 缺口描述
- 建议补充方式（访谈/文档/观察）
- 优先级

### 7.4 逻辑冲突审计

发现内容冲突时：
1. 记录冲突双方的来源
2. 分析冲突原因（版本差异/部门差异/场景差异）
3. 标注待决策项
4. 提交管理层裁决

## 八、执行纪律

### 8.1 必须遵守

1. 每次开始工作前，必须先阅读本规则文档
2. 不得自行修改本规则
3. 不得删减源材料中的有价值内容
4. 不得使用"待补充"等占位内容
5. 每个任务完成后必须更新进度记录

### 8.2 禁止事项

1. 禁止为了简洁而牺牲完整性
2. 禁止跳过流程中的任何步骤
3. 禁止使用模糊的、不可执行的描述
4. 禁止在未确认的情况下自行决定重大事项

### 8.3 状态重置协议

每完成一个5-5批次后，执行"逻辑清零"：
1. 确认萃取报告已保存
2. 生成《批次索引摘要》（不超过500字）
3. 清除本批次原始文本细节
4. 仅携带《规则.md》和《索引摘要》进入新批次
