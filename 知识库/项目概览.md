## 📊 项目概览

### 项目名称
**斯坦星球AI学院 - 企业级知识库建设**

### 项目目标

```
输入 → 处理 → 输出 → 应用
 ↓      ↓      ↓      ↓
原始   标准  知识   AI系统
素材   化    块     支撑
150+  文件  1000+  3个
```

**具体目标**：
- ✅ 收集150+份企业知识（决策、管理、执行、培训、课程）
- ✅ 转换为1000+个原子知识块
- ✅ 建立完整的UID索引和标签体系
- ✅ 支撑3个AI系统（Socratic Coach、场景模拟、问答系统）
- ✅ 为全员提供统一的知识查询入口

### 核心价值

| 对象 | 价值 |
|------|------|
| **新员工** | 快速熟悉公司，加速融入 |
| **销售** | 统一的话术库，随时查询 |
| **教师** | 课程体系透明，教学有标准 |
| **管理** | 数据一致，决策有依据 |
| **AI系统** | 知识充分，回答更准确 |

## 👥 角色与职责

### 1. 文档提供者（Content Provider）

**谁**：各部门员工

**职责**：
- 提供部门相关的知识文档
- 确保信息准确和完整
- 配合审核反馈

**权限**：
- 提交文档到指定目录
- 查看自己提交的文档状态

**奖励**：
- 贡献者排名
- 月度表彰
- 年度奖金

### 2. 审核员（Reviewer）

**谁**：部门专家、管理层代表

**职责**：
- 审核提交的文档
- 检查准确性和完整性
- 处理冲突和问题

**权限**：
- 审核和批准文档
- 要求修改和补充
- 标注冲突

**分布**：
- 各部门1-2名主审核员
- 专业领域审核员
- 总审核员1名

### 3. 知识库管理员（KB Admin）

**谁**：运营/技术团队

**职责**：
- 整体规划和推进
- 工具和流程管理
- 索引和元数据管理
- 冲突解决

**权限**：
- 所有操作权限
- 系统配置权限

### 4. 使用者（User）

**谁**：全体员工

**职责**：
- 学习和使用知识
- 提供反馈
- 提出建议

**权限**：
- 查询和阅读（权限管理实施后）
- 发起搜索请求
