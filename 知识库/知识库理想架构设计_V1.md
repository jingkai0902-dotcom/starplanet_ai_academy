## 一、设计理念

### 1.1 当前状态分析

**已完成的工作：**
| 模块 | 课程数 | 萃取状态 | 质量评估 |
|------|--------|----------|----------|
| STEM课程 | 133节 | ✅ 深度萃取 | 10/10 标杆级 |
| CODE课程 | 120节 | ✅ 原子化萃取 | 10/10 标杆级 |
| PythonAI课程 | 109节 | ✅ 原子化萃取 | 10/10 标杆级 |
| 品牌知识 | - | ✅ 深度萃取 | 10/10 标杆级 |
| **合计** | **362节** | | |

**当前问题：**
| 问题 | 表现 |
|------|------|
| 内容分散 | 萃取报告按"模块"组织，而非按"使用场景"组织 |
| 缺少知识点数据库 | 无法回答"循环语句在哪些课程出现？" |
| 缺少螺旋进阶图 | 无法追溯"变量概念从CODE到Python如何进阶" |
| 缺少项目案例库 | 各课程的项目作品分散在各萃取报告中 |
| 硬件知识分散 | ESP32/传感器知识分布在STEM、CODE、PythonAI多处 |

### 1.2 理想状态

**三个使用场景必须能回答：**

| 场景 | 问题示例 | 知识库应能提供 |
|------|---------|---------------|
| 教师备课 | "CODE1-2-3这节课怎么上？" | 完整5E流程 + 具体积木/代码 + 教学话术 |
| 家长沟通 | "孩子学完STEM小班能做什么？" | 按阶段的能力目标 + 项目作品展示 |
| 课程规划 | "循环语句从哪里开始教？怎么进阶？" | 跨课程体系的知识点追溯图 |
| 硬件选型 | "ESP32在哪些课程用到？" | 硬件使用汇总 + 接线图 + 代码模板 |

### 1.3 内容分层原则

```
┌─────────────────────────────────────────────────────┐
│  第1层：导航层（索引、目录、快速入口）               │  ✅ 已有
├─────────────────────────────────────────────────────┤
│  第2层：框架层（教学方法、5E流程、评估标准）         │  ✅ 已有
├─────────────────────────────────────────────────────┤
│  第3层：内容层（具体积木、代码、硬件接线）           │  ✅ 已有（萃取报告）
├─────────────────────────────────────────────────────┤
│  第4层：原子层（单个知识点，可被检索和关联）         │  ⚠️ 部分缺失
├─────────────────────────────────────────────────────┤
│  第5层：关联层（跨课程进阶路线、知识点复现）         │  ❌ 缺失
└─────────────────────────────────────────────────────┘
```

## 三、与励步知识库的差异

### 3.1 核心差异

| 维度 | 励步知识库 | 斯坦星球知识库 |
|------|-----------|---------------|
| **课程类型** | 英语教学（词汇、句型、语法） | STEM/编程教学（概念、代码、硬件） |
| **知识点粒度** | 单词、句型（可枚举） | 编程概念、代码模式（需抽象） |
| **螺旋进阶** | 同一单词在不同级别复现 | 同一概念在不同语言/平台复现 |
| **实践载体** | 对话、阅读、写作 | 项目、作品、硬件 |
| **评估方式** | 听说读写能力 | 项目完成度、代码质量 |

### 3.2 斯坦星球特有需求

| 需求 | 说明 |
|------|------|
| **硬件知识库** | ESP32、传感器、执行器的跨课程汇总 |
| **代码模式库** | 常用代码片段、设计模式的汇总 |
| **项目案例库** | 各课程项目作品的集中展示 |
| **跨语言进阶** | Scratch→Python→C++的概念映射 |
| **软硬结合** | 纯软件课程与硬件课程的知识关联 |

### 4.1 编程概念库示例

```markdown
# 循环结构进阶路线

## 概念定义
循环结构：让程序重复执行某段代码的控制结构

## 跨课程进阶

### CODE阶段（图形化）
| 级别 | 积木 | 示例 | UID |
|------|------|------|-----|
| CODE1-M1 | 重复10次 | 让角色走10步 | CODE1-1-3-C001 |
| CODE1-M2 | 重复直到 | 碰到边缘停止 | CODE1-2-5-C001 |
| CODE2-M1 | 嵌套循环 | 画正方形网格 | CODE2-1-4-C001 |

### PythonAI阶段（代码）
| 级别 | 语法 | 示例 | UID |
|------|------|------|-----|
| L1-00 | for i in range() | 打印1-10 | PYAI-10-03-C001 |
| L1-00 | while条件 | 猜数字游戏 | PYAI-10-05-C001 |
| L1-03 | 列表遍历 | for item in list | PYAI-13-02-C001 |
| L2-02 | 游戏主循环 | while running | PYAI-22-02-C001 |

### 概念映射
| Scratch | Python | 说明 |
|---------|--------|------|
| 重复10次 | for i in range(10) | 固定次数循环 |
| 重复直到 | while not condition | 条件循环 |
| 永远重复 | while True | 无限循环 |

## 教学建议
- CODE阶段：强调"重复"的直观理解
- Python阶段：引入"迭代"的抽象概念
- 进阶关键：从"次数"到"条件"的思维转换
```

### 5.2 整合原则

1. **不压缩**：萃取报告中的代码、话术、流程全部保留
2. **重组织**：按课程/模块/课时三级拆分
3. **加索引**：每个知识点有UID，可被检索和关联
4. **建关联**：同一概念在不同课程的出现要互相链接
5. **提取汇总**：硬件、编程概念、项目单独汇总

| 优先级 | 任务 | 原因 |
