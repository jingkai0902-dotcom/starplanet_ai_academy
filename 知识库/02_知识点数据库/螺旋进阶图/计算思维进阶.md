## 什么是计算思维？

计算思维（Computational Thinking）是一种解决问题的思维方式，包含四大核心能力：

```
┌─────────────────────────────────────────────────────────┐
│                     计算思维                             │
├─────────────┬─────────────┬─────────────┬───────────────┤
│   分解       │  模式识别    │   抽象      │   算法设计    │
│Decomposition│  Pattern    │ Abstraction │  Algorithm    │
│             │ Recognition │             │   Design      │
├─────────────┼─────────────┼─────────────┼───────────────┤
│ 把大问题    │ 发现规律    │ 提取本质    │ 设计步骤      │
│ 拆成小问题  │ 找出相似    │ 忽略细节    │ 解决问题      │
└─────────────┴─────────────┴─────────────┴───────────────┘
```

# 冒泡排序算法
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

# 二分查找算法
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

## 五、四大能力综合应用

### 5.1 解决问题的完整流程

```
遇到问题
    ↓
【分解】把大问题拆成小问题
    ↓
【模式识别】找出相似的部分
    ↓
【抽象】提取本质，忽略细节
    ↓
【算法设计】设计解决步骤
    ↓
实现解决方案
```

### 5.2 综合应用示例：贪食蛇游戏

```
【分解】
├── 蛇的移动
├── 食物生成
├── 碰撞检测
└── 计分系统

【模式识别】
├── 蛇身是一串坐标 → 用列表存储
├── 每次移动都是"加头去尾" → 提取成函数
└── 碰撞检测都是"坐标比较" → 复用代码

【抽象】
├── 蛇 → Snake类（位置、方向、长度）
├── 食物 → Food类（位置）
└── 游戏 → Game类（蛇、食物、得分）

【算法设计】
├── 移动算法：新头 = 旧头 + 方向
├── 吃食物：不去尾，蛇变长
├── 碰撞检测：头在身体列表中？
└── 游戏循环：输入→更新→渲染
```

## 八、教学建议

### 8.1 各能力培养重点

| 能力 | 培养重点 | 常用方法 |
|------|---------|---------|
| 分解 | 任务拆解习惯 | 先问"分成几步" |
| 模式识别 | 观察比较习惯 | 先问"有什么相同" |
| 抽象 | 提炼本质习惯 | 先问"最重要的是什么" |
| 算法设计 | 步骤规划习惯 | 先问"怎么一步步做" |

### 8.2 常见教学难点

| 难点 | 解决方案 |
|------|---------|
| 不会分解 | 提供分解模板，逐步引导 |
| 找不到模式 | 并排展示相似案例 |
| 抽象困难 | 从具体到抽象，多举例子 |
| 算法混乱 | 先画流程图，再写代码 |
