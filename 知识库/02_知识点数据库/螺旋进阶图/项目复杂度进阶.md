## 一、STEM小班阶段：单一功能（3-4岁）

### 1.1 项目特点

- 单一功能，一个输入或一个输出
- 机械结构为主
- 观察现象，理解因果

### 1.2 典型项目

| 项目 | 复杂度 | 涉及概念 | 学习目标 |
|------|--------|---------|---------|
| 风车 | ⭐ | 齿轮传动 | 理解力的传递 |
| 跷跷板 | ⭐ | 杠杆原理 | 理解平衡 |
| 滑轮吊车 | ⭐ | 滑轮 | 理解省力 |

### 1.3 项目结构

```
输入 → 输出
风吹 → 风车转
按压 → 跷跷板动
拉绳 → 重物升起
```

### 1.4 教学话术

> "我们做了一个会动的风车！风一吹，它就转起来了。这就是我们的第一个作品。"

## 二、STEM中班阶段：简单控制（4-5岁）

### 2.1 项目特点

- 简单的输入-输出控制
- 引入电子元件
- 理解"控制"概念

### 2.2 典型项目

| 项目 | 复杂度 | 涉及概念 | 学习目标 |
|------|--------|---------|---------|
| 电动风扇 | ⭐⭐ | 马达控制 | 理解电机 |
| 按钮灯 | ⭐⭐ | 开关控制 | 理解输入输出 |
| 声控灯 | ⭐⭐ | 传感器 | 理解感知 |

### 2.3 项目结构

```
输入 → 控制 → 输出
按钮 → 电路 → LED亮
声音 → 传感器 → 灯亮
```

## 三、STEM大班阶段：组合功能（5-6岁）

### 3.1 项目特点

- 多个输入或输出
- 简单的条件判断
- 传感器+执行器组合

### 3.2 典型项目

| 项目 | 复杂度 | 涉及概念 | 学习目标 |
|------|--------|---------|---------|
| 智能风扇 | ⭐⭐⭐ | 温度传感器+马达 | 自动控制 |
| 红绿灯 | ⭐⭐⭐ | 顺序控制 | 时序逻辑 |
| 避障小车 | ⭐⭐⭐ | 超声波+马达 | 反馈控制 |

### 3.3 项目结构

```
传感器 → 判断 → 执行器
温度高 → 如果>30° → 风扇开
距离近 → 如果<20cm → 后退
```

### 3.4 教学话术

> "我们的智能风扇会'思考'了！它感觉到热，就自己开；感觉到凉，就自己关。这就是智能！"

### 6.3 项目结构

```python
# 游戏类结构
class Game:
    def __init__(self):
        self.player = Player()
        self.enemies = []
        self.score = 0
        self.running = True

    def update(self):
        self.player.update()
        for enemy in self.enemies:
            enemy.update()
        self.check_collisions()

    def draw(self):
        self.player.draw()
        for enemy in self.enemies:
            enemy.draw()

    def run(self):
        while self.running:
            self.handle_events()
            self.update()
            self.draw()
```

### 6.4 Python代码示例

```python
import pygame

class Snake:
    def __init__(self):
        self.body = [(100, 100), (90, 100), (80, 100)]
        self.direction = (10, 0)

    def move(self):
        head = self.body[0]
        new_head = (head[0] + self.direction[0],
                    head[1] + self.direction[1])
        self.body.insert(0, new_head)
        self.body.pop()

    def grow(self):
        # 不删除尾部，蛇变长
        head = self.body[0]
        new_head = (head[0] + self.direction[0],
                    head[1] + self.direction[1])
        self.body.insert(0, new_head)

    def check_collision(self):
        head = self.body[0]
        # 撞墙
        if head[0] < 0 or head[0] >= 400 or head[1] < 0 or head[1] >= 400:
            return True
        # 撞自己
        if head in self.body[1:]:
            return True
        return False
```

## 八、复杂度评估维度

### 8.1 评估标准

| 维度 | 低复杂度 | 中复杂度 | 高复杂度 |
|------|---------|---------|---------|
| 角色数量 | 1-2个 | 3-5个 | 5+个 |
| 交互类型 | 单一输入 | 多种输入 | 多模态 |
| 数据结构 | 变量 | 列表 | 类/对象 |
| 控制流程 | 顺序 | 分支+循环 | 状态机 |
| 模块数量 | 1个 | 2-3个 | 4+个 |

### 8.2 复杂度星级

```
⭐      - 单一功能，1-2个概念
⭐⭐    - 简单交互，3-4个概念
⭐⭐⭐  - 完整游戏，5-6个概念
⭐⭐⭐⭐ - 复杂系统，7-8个概念
⭐⭐⭐⭐⭐- AI集成，9+个概念
```

## 九、教学建议

### 9.1 项目选择原则

1. **循序渐进**：从低复杂度开始，逐步增加
2. **兴趣驱动**：选择学生感兴趣的主题
3. **概念覆盖**：确保涵盖目标知识点
4. **可完成性**：在课时内能完成核心功能

### 9.2 常见教学难点

| 难点 | 解决方案 |
|------|---------|
| 项目太大 | 分解为多个小任务 |
| 概念太多 | 先完成核心功能，再扩展 |
| 调试困难 | 分模块测试，逐步集成 |
| 学生差异 | 提供基础版和进阶版 |
