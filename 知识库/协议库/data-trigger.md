# 数据驱动决策触发器协议 V1.0

> 协议编号：7.14
> 创建日期：2026-01-24
> 适用场景：生成运营执行手册时

---

## 一、核心原则

**AI在生成运营执行手册时，必须包含"如果指标低于X%，则执行Y预案"的逻辑分支，实现数据驱动的自动化决策支持。**

---

## 二、定义与价值

| 概念 | 定义 | 价值 |
|------|------|------|
| **决策触发器** | 基于数据阈值自动触发预案的逻辑规则 | 实现数据驱动的管理 |
| **阈值预警** | 当指标低于/高于阈值时发出预警 | 提前发现问题 |
| **预案关联** | 将异常指标与解决方案自动关联 | 快速响应问题 |

---

## 三、标准触发器格式

**每个运营指标必须配套以下触发器结构：**

```markdown
### 指标：[指标名称]

**指标定义**
- 计算公式：[公式]
- 统计周期：[周期]
- 目标值：[目标]

**决策触发器**

| 阈值区间 | 状态 | 触发动作 | 关联预案 |
|---------|------|---------|---------|
| ≥[目标值] | 🟢 健康 | 维持现状，总结经验 | - |
| [警戒值]-[目标值] | 🟡 预警 | 启动诊断，制定改进计划 | [预案A] |
| <[警戒值] | 🔴 危险 | 立即干预，启动应急预案 | [预案B] |

**触发后执行流程**
```
IF 指标 < [警戒值] THEN
    1. 自动调取《高频问题解决库》中的[对应诊断方案]
    2. 通知[责任人]
    3. 启动[应急预案]
    4. 设置[复查周期]
END IF
```

**关联文档**
- 诊断方案：[文档路径]
- 应急预案：[文档路径]
- 复盘模板：[文档路径]
```

## 四、核心运营指标触发器示例

```markdown
### 指标：DEMO课转化率

**指标定义**
- 计算公式：签单数 / DEMO课数 × 100%
- 统计周期：周
- 目标值：50%

**决策触发器**

| 阈值区间 | 状态 | 触发动作 | 关联预案 |
|---------|------|---------|---------|
| ≥50% | 🟢 健康 | 维持现状，提炼成功经验 | - |
| 30%-50% | 🟡 预警 | 启动诊断，分析流失环节 | DEMO转化诊断方案 |
| <30% | 🔴 危险 | 立即干预，暂停新增DEMO | DEMO转化应急预案 |

**触发后执行流程**
```
IF DEMO转化率 < 30% THEN
    1. 自动调取《高频问题解决库》→ "DEMO课转化率低"诊断方案
    2. 通知销售主管和校区负责人
    3. 启动"DEMO课质量提升专项"
    4. 每日复查，连续3天达标后解除预警
END IF
```

**诊断方案（自动调取）**

**可能原因清单**
| 原因类别 | 具体原因 | 诊断方法 | 解决方案 |
|---------|---------|---------|---------|
| 客户质量 | 名单质量差，意向不明确 | 分析名单来源转化率 | 优化名单筛选标准 |
| DEMO课质量 | 教师表现不佳 | 听课评估 | 教师培训/更换 |
| 咨询能力 | 销售话术不到位 | 录音分析 | 话术培训/陪访 |
| 价格异议 | 价格敏感度高 | 异议记录分析 | 优化价值塑造话术 |
| 竞品影响 | 竞品促销活动 | 市场调研 | 差异化话术强化 |

**关联文档**
- 诊断方案：`管理层专区/2-问题解决库/高频问题解决库.md` → "DEMO课转化率低"
- 应急预案：`管理层专区/3-运营执行手册/应急预案/DEMO转化应急预案.md`
- 复盘模板：`管理层专区/1-工具模板库/复盘模板/DEMO转化复盘.md`

`#管理层` `#数据驱动`
[Source ID: 触发器-DEMO转化率-V1]
```

## 五、常用指标触发器配置

| 指标 | 目标值 | 警戒值 | 危险值 | 统计周期 |
|------|--------|--------|--------|---------|
| DEMO转化率 | ≥50% | 30%-50% | <30% | 周 |
| 续费率 | ≥85% | 70%-85% | <70% | 月 |
| 到访率 | ≥60% | 40%-60% | <40% | 周 |
| 满意度 | ≥90% | 80%-90% | <80% | 月 |
| 流失率 | ≤5% | 5%-10% | >10% | 月 |

## 六、检查清单

**AI在生成运营执行手册时，必须检查：**

- [ ] 每个关键指标是否配套了决策触发器？
- [ ] 触发器是否包含三级阈值��健康/预警/危险）？
- [ ] 每个阈值是否关联了具体的触发动作？
- [ ] 是否关联了《高频问题解决库》中的诊断方案？
- [ ] 是否提供了触发后执行流程（伪代码）？
- [ ] 是否标注了责任人和复查周期？
