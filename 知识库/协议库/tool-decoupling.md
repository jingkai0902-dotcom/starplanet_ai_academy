# 管理工具解耦协议 V1.0

> 协议编号：7.10
> 创建日期：2026-01-24
> 适用场景：从深度报告中提取管理工具时

---

## 一、核心原则

**AI在从深度报告中提取管理工具（如DEMO执行清单）时，必须将其转化为独立的、可交互的Checklist格式，而不是长篇累牍的文字描述，确保管理者在手机端也能"即拿即用"。**

---

## 二、定义与价值

| 概念 | 定义 | 价值 |
|------|------|------|
| **工具解耦** | 将嵌入在长文档中的管理工具提取为独立文件 | 便于单独调用和更新 |
| **Checklist化** | 将流程描述转化为可勾选的检查清单 | 支持移动端使用 |
| **即拿即用** | 无需阅读上下文即可直接使用 | 提升执行效率 |

---

## 三、需要解耦的管理工具类型

| 工具类型 | 示例 | 解耦后格式 |
|---------|------|-----------|
| **执行清单** | DEMO课执行清单、家长会准备清单 | Checklist |
| **话术卡片** | 异议处理话术、续费话术 | 卡片式（场景+话术） |
| **流程图** | 853续费流程、咨询转化流程 | 步骤式Checklist |
| **评估表** | 教师评课表、学员评估表 | 表格式Checklist |
| **数据模板** | 周报模板、月报模板 | 填空式模板 |

---

## 四、Checklist格式标准

**移动端友好的Checklist必须满足：**

| 要求 | 标准 | 说明 |
|------|------|------|
| **单项简短** | 每项≤20字 | 一眼能看完 |
| **层级清晰** | 最多2级嵌套 | 避免复杂层级 |
| **可勾选** | 使用`- [ ]`格式 | 支持交互 |
| **分组明确** | 按阶段/类别分组 | 便于定位 |
| **时间标注** | 标注预计时间 | 便于时间管理 |
| **独立成文** | 无需上下文 | 可单独使用 |

---

## 五、工具解耦模板

**原始文档中的嵌入式工具：**
```markdown
### DEMO课执行流程

DEMO课是招生转化的关键环节。在DEMO课开始前，销售顾问需要完成以下准备工作：首先要确认家长和孩子的到访时间，提前15分钟到达教室进行环境检查，包括空调温度、灯光、教具摆放等。然后要准备好签到表、课程介绍资料、报名表等物料...
```

**解耦后的独立Checklist：**
```markdown
# DEMO课执行清单

## 文档信息
| 项目 | 内容 |
|------|------|
| **工具名称** | DEMO课执行清单 |
| **适用角色** | 销售顾问 |
| **使用场景** | DEMO课当天 |
| **预计总时长** | 90分钟 |
| **版本** | V1.0 |
| **最后更新** | 2026-01-24 |

---

## 一、课前准备（提前30分钟）

### 1.1 确认到访（提前1天）
- [ ] 电话/微信确认到访时间
- [ ] 确认孩子姓名、年龄、英语基础
- [ ] 确认家长关注点（效果/价格/距离）
- [ ] 发送校区地址和停车指引

### 1.2 环境检查（提前15分钟）
- [ ] 空调温度调至24-26度
- [ ] 灯光明亮、无闪烁
- [ ] 教具摆放整齐
- [ ] 白板/投影正常
- [ ] 背景音乐播放

### 1.3 物料准备
- [ ] 签到表（打印）
- [ ] 课程介绍册（2份）
- [ ] 报名表（2份）
- [ ] 小礼物（贴纸/气球）
- [ ] 饮用水（家长用）

---

## 二、课中观察（45分钟）

### 2.1 孩子表现记录
- [ ] 参与度（积极/一般/被动）
- [ ] 语言输出（主动/跟读/沉默）
- [ ] 与老师互动情况
- [ ] 与其他孩子互动情况

### 2.2 家长反应观察
- [ ] 表情变化（满意/担忧/无感）
- [ ] 是否拍照/录像
- [ ] 是否主动提问
- [ ] 关注点记录

---

## 三、课后咨询（30分钟）

### 3.1 反馈收集
- [ ] 询问孩子感受
- [ ] 询问家长观感
- [ ] 记录家长问题

### 3.2 价值呈现
- [ ] 展示课程体系
- [ ] 对比分析（如需）
- [ ] 解答疑问

### 3.3 促成签约
- [ ] 介绍优惠政策
- [ ] 处理异议
- [ ] 引导签约/预约下次

---

`#管理层` `#工具模板`
```

---

## 六、检查清单

**AI在萃取管理层内容时，必须检查：**

- [ ] 是否识别了嵌入在长文档中的管理工具？
- [ ] 识别到的工具是否已解耦为独立文件？
- [ ] 解耦后的工具是否符合Checklist格式标准？
- [ ] 每项是否≤20字？
- [ ] 是否标注了预计时间？
- [ ] 是否可以脱离上下文独立使用？
- [ ] 是否已存入`管理层专区/1-工具模板库/`？
