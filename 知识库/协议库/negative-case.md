# 7.6 负面案例萃取协议 (Negative Case Extraction Protocol)

> 来源：知识库萃取标准与执行规则.md V3.5
> **核心原则：主动搜寻"学生常见错误"和"教学误区"，将其转化为《教师考核题库》中的纠错题，提升培训针对性。**

## 负面案例的定义与价值

| 类型 | 定义 | 教研价值 |
|------|------|---------|
| **学生常见错误** | 学生在学习过程中反复出现的典型错误 | 帮助教师预判问题、提前干预 |
| **教学误区** | 教师在教学中容易犯的错误或低效做法 | 帮助新教师避坑、提升教学质量 |
| **失败案例** | 导致学生流失、家长投诉的真实案例 | 提供反面教材、强化风险意识 |
| **低效做法** | 看似正确但效果不佳的教学方法 | 纠正惯性思维、优化教学策略 |

## 负面案例搜寻关键词

**AI在萃取时必须主动搜寻以下关键词/模式：**

| 搜寻维度 | 关键词/模式 | 示例 |
|---------|-----------|------|
| **错误类型** | "常见错误"、"容易犯的错"、"误区"、"不要"、"禁止"、"避免" | "学生常见的发音错误" |
| **失败信号** | "失败"、"流失"、"投诉"、"退费"、"不满意" | "导致家长投诉的案例" |
| **对比信号** | "错误做法 vs 正确做法"、"低效 vs 高效"、"新手 vs 老手" | "新教师常犯的5个错误" |
| **纠正信号** | "纠正"、"改正"、"调整"、"优化"、"改进" | "如何纠正学生的拼读错误" |
| **警示信号** | "注意"、"警惕"、"小心"、"切忌"、"千万不要" | "课堂管理中切忌的3件事" |

## 负面案例萃取模板

```markdown
### 负面案例：[案例标题]

**案例类型**：[学生常见错误/教学误区/失败案例/低效做法]
**适用场景**：[具体教学场景/级别]
**影响程度**：[高/中/低]

**错误表现**
[具体描述错误的表现形式]

**错误原因分析**
1. [原因1]
2. [原因2]
3. [原因3]

**正确做法**
[对比说明正确的做法]

**预防措施**
- [措施1]
- [措施2]

**转化为纠错题**
> **题目**：以下哪种做法是错误的？
> A. [正确做法]
> B. [错误做法] ✓
> C. [正确做法]
> D. [正确做法]
>
> **答案解析**：[解释为什么B是错误的，正确做法应该是什么]

`#执行层` `#测评项`
[Source ID: xxxx]
```

## 负面案例与考核题库的映射规则

**每个��面案例必须转化为至少一道考核题：**

| 负面案例类型 | 题型转化 | 考核目标 |
|-------------|---------|---------|
| **学生常见错误** | 情景判断题 | 考核教师能否识别并纠正学生错误 |
| **教学误区** | 单选/多选题 | 考核教师是否了解正确做法 |
| **失败案例** | 案例分析题 | 考核教师的问题分析和应对能力 |
| **低效做法** | 对比选择题 | 考核教师能否区分高效与低效做法 |

## 题目难度分级

| 难度 | 适用对象 | 题目特点 |
|------|---------|---------|
| **初级** | 新教师入职考核 | 明显的对错判断 |
| **中级** | 在职教师季度考核 | 需要分析原因 |
| **高级** | 教学主管/骨干教师 | 需要综合判断和策略制定 |

## 负面案例萃取检查清单

**每批次萃取结束时，必须检查：**

- [ ] 是否主动搜寻了负面案例相关关键词？
- [ ] 识别到的负面案例是否已按模板结构化？
- [ ] 每个负面案例是否已转化为考核题？
- [ ] 考核题是否已标注难度级别？
- [ ] 是否已更新《教师考核题库》？
