# 7.5 逻辑冲突审计 (Conflict Audit Protocol)

> 来源：知识库萃取标准与执行规则.md V3.5
> **当不同来源对同一内容描述不一致时，AI必须标注冲突，交由用户（架构师）做最终决策，禁止自行取舍。**

## 冲突识别标准

| 冲突类型 | 识别标准 | 处理方式 |
|---------|---------|---------|
| **流程冲突** | 同一流程的步骤数量或顺序不同 | 标注所有版本，请求决策 |
| **数据冲突** | 同一指标的数值不同 | 标注所有数值及来源，请求决策 |
| **定义冲突** | 同一概念的定义不同 | 标注所有定义，请求决策 |
| **话术冲突** | 同一场景的推荐话术不同 | 标注所有版本，请求决策 |
| **体系冲突** | 新旧课程体系内容混杂 | 标注来源体系，请求决策 |

## 冲突标注格式

**当发现冲突时，必须按以下格式标注：**

```markdown
---
## ⚠️ 逻辑冲突审计

### 冲突项：[冲突主题]

**冲突类型**：[流程冲突/数据冲突/定义冲突/话术冲突/体系冲突]

**版本A**（来源：[Source ID]）：
> [具体内容]

**版本B**（来源：[Source ID]）：
> [具体内容]

**版本C**（来源：[Source ID]，如有）：
> [具体内容]

**AI分析**：
- 版本A的特点：[分析]
- 版本B的特点：[分析]
- 可能的原因：[分析，如新旧版本、不同场景等]

**请架构师决策**：
- [ ] 采用版本A
- [ ] 采用版本B
- [ ] 合并整合
- [ ] 全部保留并标注适用场景
- [ ] 其他：___________

---
```

## 冲突审计示例

```markdown
## ⚠️ 逻辑冲突审计

### 冲突项：Story五步法的具体步骤

**冲突类型**：流程冲突

**版本A**（来源：Stage1-4逻辑-15）：
> Story五步法：
> 1. Pre-reading（读前预测）
> 2. First Reading（首次阅读）
> 3. Second Reading（精读）
> 4. Post-reading（读后活动）
> 5. Extension（拓展延伸）

**版本B**（来源：阅读教学-08）：
> Story教学流程：
> 1. Lead-in（导入）
> 2. Prediction（预测）
> 3. Reading（阅读理解）
> 4. Language Focus（语言聚焦）
> 5. Production（产出）

**AI分析**：
- 版本A侧重阅读技能培养，强调多次阅读
- 版本B侧重语言教学，强调语言点和产出
- 可能的原因：版本A来自旧课程体系（Stage 1-4），版本B来自现行课程体系

**请架构师决策**：
- [ ] 采用版本A
- [x] 采用版本B（现行课程体系）
- [ ] 合并整合
- [ ] 全部保留并标注适用场景
- [ ] 其他：___________
```

## 体系冲突特别处理

**针对新旧课程体系冲突（Stage 1-4 vs K1-G3-SS），执行以下规则：**

| 内容类型 | 处理方式 |
|---------|---------|
| **课程逻辑/体系架构** | 以K1-G3-SS为准，剔除Stage 1-4的体系描述 |
| **教学方法/技巧** | 保留有价值的方法，但需验证是否与现行体系兼容 |
| **具体话术/案例** | 保留，但标注来源体系，供参考使用 |
| **流程/步骤** | 标注冲突，请求架构师决策 |

**旧体系内容标注格式：**
```markdown
> ⚠️ **来源提醒**：以下内容来自旧课程体系（Stage 1-4），仅供参考。
> 如与现行K1-G3-SS体系有冲突，以现行体系为准。
```
