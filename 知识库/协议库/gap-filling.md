# 7.4 动态补漏指令 (Gap-Filling Protocol)

> 来源：知识库萃取标准与执行规则.md V3.5
> **当AI识别出知识缺口时，不应只是记录，必须自动生成具体的补漏执行任务。**

## 缺口识别标准

| 缺口类型 | 识别标准 | 示例 |
|---------|---------|------|
| **内容缺失** | 某主题在所有来源中完全没有涉及 | 负面案例库、失败教训 |
| **深度不足** | 有提及但缺乏具体操作细节 | 只有理论没有话术 |
| **场景缺失** | 缺少特定场景的应对方案 | 城市差异化策略、特殊客户类型 |
| **数据缺失** | 缺少支撑决策的数据 | 转化率、续费率基准值 |
| **案例缺失** | 缺少真实案例支撑 | 成功/失败案例 |

## 补漏指令生成规范

**当识别出缺口时，必须按以下格式生成补漏任务：**

```markdown
---
## 知识缺口与补漏任务

### 缺口1：[缺口名称]

**缺口类型**：[内容缺失/深度不足/场景缺失/数据缺失/案例缺失]
**影响范围**：[影响哪些文档/哪些用户]
**紧急程度**：[P0紧急/P1重要/P2一般]

**补漏方式**：
- [ ] 搜索现有笔记本：[具体搜索关键词]
- [ ] 访谈对象：[具体人员/岗位]
- [ ] 数据收集：[具体数据项]
- [ ] 外部资料：[具体来源]

**访谈提纲**（如需访谈）：
1. [具体问题1]
2. [具体问题2]
3. [具体问题3]

**预期产出**：
- [具体产出物1]
- [具体产出物2]

---
```

## 补漏任务示例

### 示例1：负面案例库

```markdown
### 缺口1：负面案例库

**缺口类型**：案例缺失
**影响范围**：销售培训、新人避坑指南
**紧急程度**：P1重要

**补漏方式**：
- [ ] 访谈对象：销售主管、资深销售顾问
- [ ] 数据收集：近6个月丢单记录

**访谈提纲**：
1. 最近半年印象最深的丢单案例是什么？当时是怎么处理的？
2. 新人最容易犯的3个错误是什么？
3. 有没有因为某句话说错导致客户流失的案例？
4. 竞品抢单成功的案例有哪些？我们输在哪里？

**预期产出**：
- 销售负面案例库（10-15个典型案例）
- 新人避坑清单
- 竞品抢单应对策略
```

### 示例2：城市差异化策略

```markdown
### 缺口2：城市差异化策略

**缺口类型**：场景缺失
**影响范围**：市场策略、销售话术
**紧急程度**：P2一般

**补漏方式**：
- [ ] 访谈对象：各城市校区负责人
- [ ] 数据收集：各城市客户画像、竞品分布

**访谈提纲**：
1. 您所在城市的家长最关心什么？（价格/效果/品牌/距离）
2. 当地主要竞品是谁？他们的优势是什么？
3. 您在当地最有效的获客方式是什么？
4. 有没有当地特有的异议需要处理？

**预期产出**：
- 城市差异化策略手册
- 各城市竞品对比表
- 区域化话术库
```
