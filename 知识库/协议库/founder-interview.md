# 7.7 创始人访谈提纲生成协议 (Founder Interview Protocol)

> 来源：知识库萃取标准与执行规则.md V3.5
> **核心原则：将AI识别的"知识缺口"自动转化为"创始人访谈提纲"，通过人工干预填补AI无法从现有资料中获取的深度隐性经验。**

## 协议触发条件

**当AI识别到以下类型的知识缺口时，必须自动生成访谈提纲：**

| 缺口类型 | 触发条件 | 访谈对象建议 |
|---------|---------|-------------|
| **隐性经验缺失** | 现有资料只有"做什么"，缺少"为什么这样做" | 创始人、资深管理者 |
| **决策逻辑缺失** | 涉及战略决策但缺少决策背景和考量 | 创始人、决策层 |
| **失败教训缺失** | 只有成功案例，缺少失败案例和教训 | 创始人、一线管理者 |
| **差异化策略缺失** | 缺少与竞品差异化的深层逻辑 | 创始人、销售主管 |
| **文化理念缺失** | 缺少企业文化、价值观的深度阐释 | 创始人 |
| **历史演变缺失** | 缺少业务/流程演变的历史背景 | 创始人、资深员工 |

## 访谈提纲生成模板

```markdown
---
## 创始人访谈提纲

### 访谈信息

| 项目 | 内容 |
|------|------|
| **访谈主题** | [主题名称] |
| **知识缺口来源** | [来自哪个萃取报告/笔记本] |
| **缺口类型** | [隐性经验/决策逻辑/失败教训/差异化策略/文化理念/历史演变] |
| **建议访谈对象** | [创始人/XX主管/资深XX] |
| **预计时长** | [30分钟/60分钟/90分钟] |
| **紧急程度** | [P0紧急/P1重要/P2一般] |

### 访谈背景

**现有资料已覆盖**：
- [已有内容1]
- [已有内容2]

**识别到的知识缺口**：
- [缺口1：具体描述]
- [缺口2：具体描述]

**填补缺口的价值**：
- [价值1：对XX文档/XX培训的支撑]
- [价值2：对XX决策的支撑]

### 访谈问题清单

#### 第一部分：[主题1]（约X分钟）

1. **开放性问题**：[问题内容]
   - 追问方向：[如果回答XX，追问XX]

2. **具体化问题**：[问题内容]
   - 期望获取：[具体的信息类型]

3. **案例挖掘**：[问题内容]
   - 示例引导："能否举一个具体的例子？"

#### 第二部分：[主题2]（约X分钟）

4. **决策逻辑**：[问题内容]
   - 追问方向：[当时有哪些备选方案？为什么选择了这个？]

5. **失败教训**：[问题内容]
   - 示例引导："有没有走过弯路？当时是怎么调整的？"

6. **隐性经验**：[问题内容]
   - 追问方向：[这个经验是怎么总结出来的？]

#### 第三部分：总结与补充（约5分钟）

7. **遗漏检查**："关于[主题]，还有什么是我们没有问到但您觉得很重要的？"

8. **资料补充**："您手边有没有相关的文档、数据或案例可以分享？"

### 访谈产出预期

| 产出物 | 说明 | 目标文档 |
|--------|------|---------|
| [产出1] | [具体说明] | [将整合到哪个文档] |
| [产出2] | [具体说明] | [将整合到哪个文档] |

### 访谈后处理

- [ ] 访谈录音/笔记整理
- [ ] 关键信息结构化萃取
- [ ] 更新目标文档
- [ ] 标注Source ID（格式：访谈-[日期]-[主题]）
- [ ] 更新《深度萃取执行计划》

---
```

## 访谈提纲自动生成规则

**当萃取报告中出现以下内容时，AI必须自动生成对应的访谈提纲：**

| 萃取报告中的信号 | 自动生成的访谈主题 |
|-----------------|-------------------|
| "缺少负面案例" | 《失败案例与教训访谈提纲》 |
| "缺少决策背景" | 《战略决策逻辑访谈提纲》 |
| "缺少差异化深层逻辑" | 《竞争差异化策略访谈提纲》 |
| "缺少不同学习风格适配" | 《分层教学经验访谈提纲》 |
| "缺少历史演变" | 《业务发展历程访谈提纲》 |
| "缺少文化理念阐释" | 《企业文化与价值观访谈提纲》 |

## 访谈提纲存档规范

**访谈提纲统一存放路径**：`知识库/萃取报告/访谈提纲/`

**文件命名规范**：`访谈提纲_[主题]_[日期].md`

**示例**：`访谈提纲_Phonics教学负面案例_2026-01-23.md`
