# 高价值信息向上提纯协议 V1.0

> 协议编号：7.9
> 创建日期：2026-01-24
> 适用场景：萃取过程中发现战略价值信息时

---

## 一、核心原则

**AI在执行任何层级的萃取时，只要发现涉及"行业壁垒"、"重大成本变动"或"核心竞争优势"的信息，必须自动复制一份并打上 #决策层 标签，汇编进《决策层专区》，确保创始人和高管能看到最底层的核心洞察。**

---

## 二、定义与价值

| 概念 | 定义 | 价值 |
|------|------|------|
| **向上提纯** | 从执行层/管理层萃取中识别具有战略价值的信息，自动复制到决策层 | 确保高层不遗漏底层洞察 |
| **信息穿透** | 让一线的关键发现能直达决策层 | 打破信息层级壁垒 |
| **战略预警** | 提前识别可能影响战略的信息 | 支持前瞻性决策 |

---

## 三、强制向上提纯的信息类型

**AI在萃取过程中发现以下类型信息时，必须自动执行向上提纯：**

| 信息类型 | 识别关键词 | 战略价值 | 提纯优先级 |
|---------|-----------|---------|-----------|
| **行业壁垒** | "护城河"、"独家"、"专利"、"壁垒"、"难以复制" | 竞争优势来源 | P0 |
| **重大成本变动** | "成本上涨"、"人力成本"、"租金"、"涨价"、"降本" | 盈利能力影响 | P0 |
| **核心竞争优势** | "差异化"、"独特"、"领先"、"对标"、"超越" | 市场定位依据 | P0 |
| **客户流失信号** | "流失"、"退费"、"投诉"、"不续费"、"转竞品" | 业务风险预警 | P1 |
| **市场机会** | "新需求"、"蓝海"、"增长点"、"趋势"、"政策利好" | 扩张决策依据 | P1 |
| **人才风险** | "离职"、"挖角"、"招聘难"、"核心员工" | 组织稳定性 | P1 |
| **竞品动态** | "竞品"、"对手"、"新机构"、"价格战"、"促销" | 竞争策略调整 | P2 |
| **政策变化** | "政策"、"监管"、"合规"、"资质"、"双减" | 合规风险管理 | P2 |

---

## 四、执行流程

```
步骤1：识别高价值信息
├── 在萃取过程中扫描关键词
├── 判断信息是否符合向上提纯标准
└── 标注提纯优先级（P0/P1/P2）

步骤2：信息提纯处理
├── 提取原始信息的核心内容
├── 补充上下文背景
├── 分析战略影响
└── 提出决策建议（如有）

步骤3：双重存档
├── 原始位置保留完整内容
├── 决策层专区存入提纯版本
└── 建立双向引用链接

步骤4：汇编与推送
├── 定期汇编《决策层信息简报》
├── 按优先级排序
└── 标注信息来源和萃取日期
```

## 六、检查清单

**每批次萃取结束时，必须检查：**

- [ ] 是否扫描了所有高价值信息关键词？
- [ ] 识别到的高价值信息是否已执行向上提纯？
- [ ] 提纯信息是否已存入决策层专区？
- [ ] 是否建立了原始位置与决策层的双向引用？
- [ ] P0级信息是否已标注"需立即关注"？
